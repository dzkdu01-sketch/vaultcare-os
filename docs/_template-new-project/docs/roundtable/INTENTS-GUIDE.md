# 意图系统使用指南（子项目版）

> 版本：1.0（模板种子，允许子项目独立进化）  
> 约定：AI 每次回复开头必须输出模式标签：`[人机协作-XXX]`

---

## 新会话入口（必读顺序）

1. `docs/status/current-status.md`
2. `docs/handover.md`
3. 本文件：`docs/roundtable/INTENTS-GUIDE.md`

---

## 核心意图（建议优先使用）

| 意图 | 作用 |
|------|------|
| `/启动` | 新对话开始，自动读取上下文并输出摘要 |
| `/开发` | 开发新功能（带门禁：目标/边界/验收） |
| `/Bug` | 报告 Bug（结构化记录并写入 current-status） |
| `/圆桌` | 复杂问题讨论（v2：专家推荐→确认→发散→提取→固化） |

子项目也可以逐步补齐其他意图（如 `/测试` `/文档` `/进度` 等），但不阻塞最小可用。

---

## 开发硬门禁（v1）

进入 `/开发` 前，必须确认：
1. 目标：本轮唯一业务目标（1 句话）
2. 边界：本轮明确不做项（至少 1 条）
3. 验收：3-5 条可验证结果（至少 1 条页面行为）

---

## 圆桌 v2（最小约束）

- 圆桌开场必须先输出“专家推荐名单”，等待用户确认/增补后再进入正式讨论
- 标准流程：业务上下文注入 → 发散 → 提取 → 固化
- 必须产出：方案对比表（≥3）+ 待确认问题（≤5）+ 建议决议

详见：`docs/roundtable/roundtable-v2-protocol.md` 与 `docs/roundtable/intents/04-圆桌.md`
