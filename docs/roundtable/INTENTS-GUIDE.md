## 快速开始

### 什么是意图？

意图是**打包的人机协作流程**。你说一个词，AI 自动执行完整流程。

### 强制状态标识

新对话默认进入人机协作，AI 每次回复**开头**必须声明当前模式，格式：`[人机协作-XXX]`（如 `[人机协作-日常]`、`[人机协作-圆桌]`）。详见各意图文档。

### 上下文读取顺序（新增）

涉及跨模块流程讨论或需求澄清时，优先按以下顺序读取：

1. `docs/status/current-status.md`
2. `docs/handover.md`
3. `docs/process-registry.md`（业务-流程-功能映射）
4. 对应模块 spec / matrix / sessions

---

### 如何使用？

**方式 1：意图词**
```
/启动
/开发 供应商编辑功能
/Bug 供应商 Tab 不能编辑
```

**方式 2：自然语言**
```
新对话开始
修复供应商 Tab 只读问题
```

**方式 3：混合**
```
/启动
今天想修复供应商 Tab 的问题
```

## 使用示例

### 场景 1：新对话开始

```
用户：/启动

AI：（读取上下文，输出状态摘要）
```

### 场景 2：开发新功能

```
用户：/开发 供应商编辑功能

AI：收到。我先给默认业务草案（目标/边界/验收），你只需确认 3 点：
1. 这轮优先“稳定”还是“效率”？
2. 本轮明确不做哪些事项？
3. 哪些业务结果必须同时达成？

用户：效率优先，不改流程角色，保存后列表正确刷新且状态正确

AI：明白。先输出流程设计...
```

### 场景 3：报告 Bug

```
用户：/Bug 供应商 Tab 不能编辑

AI：收到。为了记录这个 Bug，我需要确认 3 个问题：
1. 现象：点击编辑按钮没反应，还是有报错？
2. 复现：每次都能复现吗？
3. 期望：点击后应该弹出编辑表单？

用户：点击没反应，每次都能复现，应该弹出表单

AI：（记录到 current-status.md）
```

### 场景 4：复杂问题讨论（圆桌 v2）

```
用户：/圆桌 订单审单流程优化

AI：[人机协作-圆桌] 收到。这是效率问题。

## 专家推荐
（推荐名单 + 理由 + 缺失风险）

请确认名单：[确认 / 增补：角色名 / 调整：说明]

用户：确认

AI：（发散 → 提取，由 AI 作为主力发散，保底输出 3 个截然不同的解决方向并进行红队碰撞，随后输出方案对比表 + 待确认问题 + 建议决议）
```

### 场景 5：代码审查

```
用户：/审查 @ProductList.tsx

AI：（输出审查报告，按风险等级排序）
```

## 高级功能

### 意图组合

可以连续使用多个意图：

```
用户：/启动
用户：/开发 供应商编辑功能
用户：/测试 供应商编辑功能
用户：/文档
用户：/进度
```

### 中断恢复

如果意图流程中断，可以用 `/继续` 恢复：

```
用户：/开发 供应商编辑功能
AI：收到。为了准确实现，我需要确认...
（用户离开）
（用户返回）
用户：/继续
AI：好的，我们继续。刚才在确认约束条件...
```

### 自动续航（激进型 v1）

当会话出现“上下文膨胀/目标漂移/重复澄清”时，AI 会触发**自动续航建议**（不是强制切换）：

- 触发即输出：`请新开会话并发送：/启动 + 续航包`
- 续航包包含 5 段：目标、边界、已完成证据、未完成（<=3）、下一步第一动作
- 8 轮冷却窗口内最多建议 1 次，避免频繁打断
- 正在执行命令/测试/文档落库时，建议会延后到当前动作结束后再发出

推荐执行：

```bash
/启动
（粘贴续航包）
```

不切会话时：

```bash
/继续
```

### 自动直推（v1.2，新增）

当用户表达“继续/直接做/不用确认”时，AI 默认进入**自动直推**：

- 不再每步征求确认，直接连续执行下一步
- **门禁优先**：若目标/边界/验收未闭环，自动直推不生效，必须先补齐门禁
- 仅在以下场景允许打断：
  - 外部阻塞（凭据/环境/权限）
  - 高风险不可逆操作（删库、批量删除、生产变更）
  - 目标冲突（与已确认决策冲突）
- 每轮动作结束后必须自动做三件事：
  1. 回填 `current-status` 证据
  2. 生成 `docs/handover/auto/session-*.md` 快照
  3. 更新飞轮周报/指标（`weekly-report` + `metrics-YYYY-MM.json`）

### 开发硬门禁（v1.3，新增）

进入 `/开发` 前，必须先通过以下 3 项门禁；未通过时 AI 必须拒绝进入开发：

1. 目标：本轮唯一业务目标（1 句话）
2. 边界：本轮明确不做项（至少 1 条）
3. 验收：3-5 条可验证结果（至少 1 条页面行为）

执行规则：

- AI 每轮最多追问 3 个阻断问题；
- 用户若说“继续/直接做”，但门禁未齐，AI 仍需先补门禁；
- 发现“已开发但口径不清”时，先执行飞轮纠偏，再恢复开发。
- 每次开发闭环最多回填 3 处：`docs/status/current-status.md`、`docs/quality/system-functional-matrix.md`、`docs/roundtable/consensus.md`。
- 当用户触发“新开发”时，先走圆桌式步进评估（需求评估 -> 方案提取 -> 验收草案），再进入 `/开发` 实施。
- 新开发门禁确认后，先写入待开发清单（固定文件：`docs/plans/development-backlog.md`）再开工。

### 机制冲突防护（新增）

当用户请求与协作机制冲突（如跳过门禁、文档分散写入、回填超范围）时，AI 必须先：

1. 指出冲突点；
2. 说明影响范围；
3. 给出替代方案；
4. 等用户确认后再执行。

### 自动收尾（新增，强制）

每次开发或修 Bug 完成后，AI 必须自动输出并执行：

1. 验收结果（通过/未通过 + 原因）
2. 改动清单（代码/文档）
3. 文档回填清单（最多 3 处）
4. 更新 `docs/plans/development-backlog.md` 的事项状态
5. 下一步第一动作

### 上下文高压续航（v1.2，新增）

当会话接近上下文上限时：

- AI 不等待用户要求，必须先输出“续航包 v1”
- 同步写入最新 auto handover 快照
- 给出可直接粘贴的新会话入口：

```bash
/启动
（粘贴续航包）
```

### 帮助系统

忘记意图词时，用 `/help` 查询：

```
用户：/help

AI：（输出完整意图列表）
```

```markdown

```

```
用户：//上线
```

## 意图边界说明

### `/Bug` vs `/调试`

| 意图 | 目的 | 输出 |
|------|------|------|
| `/Bug` | 报告已知问题 | 记录到 status，分配优先级 |
| `/调试` | 定位问题原因 | 找到根因，给出修复建议 |

### `/开发` vs `/重构`

| 意图 | 目的 | 输出 |
|------|------|------|
| `/开发` | 开发新功能 | 追问约束 → 生成代码 → 自审查 |
| `/重构` | 优化现有代码 | 分析问题 → 重构方案 → 执行 |

---

## 注意事项

1. **意图词不是必须的**：直接说自然语言也可以
2. **AI 会先预填再追问**：默认先给业务草案，再追问最多 3 个阻断问题
3. **可以打断**：任何时候可以打断当前流程
4. **可以跳过**：如果 AI 追问的问题你不想回答，可以说"跳过"
5. **开发前有门禁**：目标/边界/验收缺失时，AI 可拒绝直接进入 `/开发`
6. **业务优先提问**：默认不要求用户回答技术实现细节

## 与圆桌、飞轮的配合

### 三层架构

意图系统是人机协作体系的**第一层**，与圆桌头脑风暴、飞轮进化共同构成完整体系：

| 层级 | 机制 | 作用 | 使用频率 |
|------|------|------|----------|
| 第一层 | 意图系统 | 日常协作，高效执行 | 每天多次 |
| 第二层 | 圆桌头脑风暴 | 复杂问题，多专家讨论 | 每周 0-2 次 |
| 第三层 | 飞轮进化 | 持续改进，系统越用越好 | 每周/每月 |

### 意图 → 圆桌（升级）

当意图无法解决复杂问题时，升级到圆桌：

```
用户：/调试 XXX
AI：（分析问题，发现涉及多模块）
AI：这个问题涉及多个模块的协作，建议启动圆桌讨论。
是否现在启动？（是/否）

用户：是
AI：好的，启动圆桌讨论...
```

### 圆桌 → 意图（执行）

圆桌决策后，用意图执行：

```
圆桌决策：采用方案 B，重构供应商模块

用户：/开发 按方案 B 重构供应商模块
AI：收到。根据圆桌决策，方案 B 的核心是...
确认开始执行吗？（是/否）
```

### 飞轮 → 意图（优化）

飞轮机制根据使用数据优化意图：

```
飞轮数据：/调试 完成率 40%（低于平均）
   ↓
分析原因：追问步骤太多（平均 5 步）
   ↓
优化建议：简化为 3 步
   ↓
每周确认：用户同意
   ↓
执行优化：更新 /调试 意图文档
   ↓
验证效果：下周完成率提升到 75%
```

## 反馈与优化

如果意图系统有问题或建议：

1. 用 `/Bug` 报告问题
2. 用 `/圆桌` 讨论优化方案
3. 用 `/总结` 参与飞轮进化
4. 决策会写入 `consensus.md`

## 飞轮指标最小集（2026-03-04）

与圆桌 v2 和「发散-提取-固化」思路对齐的验证指标：

| 指标 | 用途 |
|------|------|
| 专家组合有效性 | 圆桌会后反馈，优化专家路由 |
| 追问轮次 | 衡量发散-提取是否高效 |
| 完成率 | 意图流程完成数/启动数 |
| 返工率 | 用户说「不对」次数/总轮次 |

详见 `docs/roundtable/roundtable-v2-protocol.md` § 三。
