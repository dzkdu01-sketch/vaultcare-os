## 这个文件夹是做什么的

这里是 Vaultcare 子项目的独立文档区，专门服务本次中台一期的需求确认、规则固化、方案设计、原型验证和后续开发准备。

当前文档区已经从“讨论稿驱动”切换到“正式需求主文档 + 支撑文档”的结构：

- 正式需求统一收口到一份主文档
- 状态机、字段字典、API、权限、原型等作为支撑文档继续保留
- 被正式文档覆盖的讨论稿统一进入归档区，保留后续回溯和改需求的可能性

---

## 当前结构

- `00-README.md`
  - 说明这个文档区是做什么的，以及当前主入口和维护规则
- `01-INDEX.md`

- `10-业务底稿/`

- `20-方案设计/`

## 维护规则

1. 根目录只放 `README` 和 `INDEX`。
2. 正式有效文档统一由 `01-INDEX.md` 维护。
3. `20-方案设计/12-一期需求说明书.md` 是当前一期需求主文档。
4. 状态机、字段字典、API、权限、原型等文档作为主文档的支撑文档继续维护。
5. 被主文档覆盖或被新版替代的文档不直接删除，统一移入 `90-归档/`。
6. 如果后续要改需求，优先修改主文档，再检查是否需要同步更新支撑文档。
7. 每份有效文档开头建议写明：日期、状态、作用、上位文档。
