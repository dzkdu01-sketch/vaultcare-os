# 意图：/重构

**别名：** `/优化` `/refactor`

**触发时机：** 优化现有代码

---

## 作用

分析代码问题，给出重构方案，提高可读性/性能/可维护性。

---

## 执行流程

```
1. 询问重构目标（可读性/性能/可维护性）
2. 分析当前代码问题
3. 给出重构方案
4. 确认后执行
5. 输出变更对比
```

---

## 重构目标选项

| 目标 | 关注点 |
|------|--------|
| 可读性 | 命名清晰、注释充分、逻辑简洁 |
| 性能 | 减少查询、优化循环、使用缓存 |
| 可维护性 | 模块化、单一职责、易扩展 |

---

## 示例

**用户：** `/重构 @ProductList.tsx`

**AI：**
```
---
## 重构分析

**重构目标：** 可读性 + 可维护性

**当前问题：**
1. 组件过长（350 行）→ 建议拆分
2. 魔法数字 768 → 提取为常量
3. 嵌套过深（4 层）→ 提前返回

**重构方案：**

### 方案 A：保守重构（推荐）
- 提取常量
- 拆分辅助函数
- 保持主结构不变

### 方案 B：激进重构
- 完全重写组件
- 引入新架构
- 风险较高

---
确认执行方案 A 吗？（是/否）
```
