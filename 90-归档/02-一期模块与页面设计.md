## 1. 文档回答什么问题

这份文档只回答 4 个问题：

1. 商品 / 分销 / 订单三个模块一期怎么定义
2. 每个模块一期必须有的核心功能是什么
3. 每个模块一期坚决不做什么
4. 每个模块对应的页面 / 工作台结构是什么

## 2. 商品模块

### 2.1 一期定位

商品模块一期不是资料仓库，而是：

**商品建档工作台 + AI 辅助内容处理 + 图片素材处理 + 可分销门禁**

### 2.2 必须有的核心功能

#### 商品建档主流程

- 统一入池
- 主商品身份
- 供应商关联（绑定供应商，标记货权类型：自营 / 外部）
- 资料整理区
- 状态流转
- 可卖商品资料包输出
- 可分销门禁

#### AI 辅助

- 原始资料解析
- 全量规范内容生成（标题 / 短描述 / 长描述 / 品类 / 标签 / 规格）
- 单字段重试（标题 / 短描述 / 长描述等，最多 3 次）
- 翻译辅助
- 文案规范化
- 单个商品优化
- 批量商品优化
- AI 结果人工确认
- 存量商品全量刷新

#### 图片素材处理

- 图片尺寸检查
- 图片质量检查
- 主图 / 附图选择
- 素材状态管理
- 批量素材检查
- 不合规提示（只提示，不自动裁切/压缩）

### 2.3 一期坚决不做

- 完整 PIM 大一统资料中心
- 全自动发布级内容工厂
- 复杂图片修图系统
- 全自动去重归并
- 多市场深度本地化内容引擎

### 2.4 商品模块页面

#### 页面 1：商品工作台首页

定位：任务池首页 + 批量处理控制台

核心区块：

- 顶部概览区
- 快速入口区
- 任务筛选区
- 任务列表区
- 行内确认区
- 异常面板

核心字段：

- 任务编号
- 商品主编码
- 标题
- 来源
- 目标市场
- 当前状态
- AI 状态
- 内容确认状态
- 素材状态
- 是否可分销
- 最后更新时间

核心按钮：

- 新建商品
- 导入资料
- 批量 AI 规范化
- 批量 AI 翻译
- 批量素材检查
- 批量确认
- 进入详情

交互要求：

- 列表页支持“行内展开确认”，不跳转专门页面
- 行内展开后展示全字段新旧对比：标题 / 短描述 / 长描述 / 品类 / 标签 / 价格 / 校验结果
- 批量 AI 处理后，运营可在列表页逐条确认、跳过或进入详情精修
- 支持筛选“待 AI 处理 / 待人工确认 / 待补资料 / 已确认未同步”等任务状态

#### 页面 2：商品工作台详情页

定位：单商品生产页

核心区块：

- 顶部状态栏
- 来源与身份区（含供应商关联和货权类型标记）
- 原始资料区
- 内容编辑区
- AI 辅助区
- 素材处理区
- 门禁区

核心按钮：

- 保存草稿
- AI 生成规范内容
- 单字段重试
- AI 翻译
- 确认身份
- 确认主图
- 提交为可分销
- 挂起

交互要求：

- AI 采用“全生成 + 单项重试”模式
- 点击“AI 生成规范内容”后，同时生成标题、短描述、长描述、品类建议、标签建议、规格字段建议
- 每个字段都需支持：查看结果、编辑、确认、重试
- 单字段重试上限为 3 次
- AI 结果一旦被运营确认，对应内容门禁可直接视为通过
- 素材区只做检查和提示，不自动处理图片
- 门禁区需明确展示：已确认字段、未确认字段、素材不合规项、当前卡住原因

## 3. 分销模块

### 3.1 一期定位

分销模块一期不是完整渠道系统，而是：

**分销站点协同模块**

### 3.2 必须有的核心功能

- 分销商主档（含关联 WhatsApp 联系账号，用于拒收任务快速定位联系渠道）
- 域名站点管理
- webhook 信息管理（双向：发布推送 webhook + 订单回传 webhook）
- 商品选品关系
- 商品到站点的发布关系
- 发布状态回执
- 失败重试
- 商品可分销校验
- 分销关键动作留痕

### 3.3 一期坚决不做

- 完整账号资产管理系统（但分销商主档保留最小化的 WhatsApp 联系账号字段）
- WhatsApp / 消息归属管理
- 分佣系统
- 渠道结算系统
- 分销 CRM
- 渠道业绩管理
- 全渠道营销分发系统

### 3.4 分销模块页面

#### 页面 3：分销商与站点管理页

定位：资产配置页

核心区块：

- 分销商列表
- 域名站点区
- webhook 配置区

核心字段：

- 分销商名称
- 市场
- 关联 WhatsApp 联系账号
- 站点名称
- 域名
- 发布 webhook 地址
- 发布 webhook 状态
- 订单回传 webhook 地址
- 订单回传 webhook 状态

核心按钮：

- 新增分销商
- 新增站点
- 配置发布 webhook
- 配置订单回传 webhook
- 测试 webhook
- 启用 / 停用

#### 页面 4：分销选品与发布页

定位：把商品发到站点的主页面

核心区块：

- 可分销商品池
- 分销商选择区
- 站点选择区
- 发布信息区
- 留痕记录区

核心字段：

- 商品主编码
- 商品标题
- 目标市场
- 分销商
- 目标站点
- 发布状态
- 失败原因

核心按钮：

- 选品
- 批量分发
- 单站发布
- 撤销
- 重试

#### 页面 5：发布状态与异常页

定位：发布监控页

核心区块：

- 异常区

核心字段：

- 商品
- 分销商
- 站点
- 域名
- 当前状态
- 失败原因
- 最近操作

核心按钮：

- 查看回执
- 重试
- 撤销
- 回退商品侧补资料

## 4. 订单模块

### 4.1 一期定位

订单模块一期不是财务系统，也不是仓储系统，而是：

**订单运营工作台**

订单有两种来源：

1. **网站订单（主流）：** WordPress 站点通过订单回传 webhook 自动推入统一订单池，直接进入"待审核"状态
2. **非网站订单（补充）：** WhatsApp / 互动消息等渠道的订单，由运营手动创建"草稿"，确认后提交"待审核"

主链路：

`来源接入 → [草稿（仅非网站渠道）] → 待审核 → 待交接 → 结果关闭`

### 4.2 必须有的核心功能

- WordPress 订单自动接入（通过站点订单回传 webhook）
- 非网站渠道手动录入入口
- 统一订单池
- 订单草稿（仅用于非网站渠道手动录入）
- 客人确认后提交待审核
- 审单工作区
- 最小化审单规则（按供应商判断是否拆单、按市场判断履约路由）
- 拆单支持（依据商品供应商归属）
- 订单交接记录（含履约路由：货盘 vs 自营仓库）
- 异常单处理
- 拒收任务
- 拒收结算标记（最小化，仅标记结算状态以支持任务关闭）
- 订单核心动作留痕

### 4.3 一期坚决不做

- 完整财务对账系统
- 完整回款系统
- 完整仓储履约系统
- 自动风控引擎
- 完整 CRM
- 复杂售后系统
- 重 BI 平台

- 回退重处理

## 5. 一期页面优先级

### 5.1 MVP 必须先做

1. 商品工作台首页
2. 商品工作台详情页
3. 分销商与站点管理页（分销选品与发布的前置依赖，必须先于或同时于页面 4 可用）
4. 分销选品与发布页
5. 订单工作台首页
6. 订单详情与处理页

### 5.2 一期后半段补齐

7. 发布状态与异常页
8. 异常与拒收任务页

---

## 6. 页面设计原则

1. 首页看任务，详情做处理。
2. 状态先于表单，门禁先于动作。
3. 异常必须独立可见。
4. AI 只做辅助，关键结果必须人工确认。
5. 分销页以域名站点和 webhook 可用性为核心。
6. 订单页以草稿、待审核、待交接为主链，而不是以详情展示为主。
