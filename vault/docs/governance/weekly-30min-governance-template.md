# 每周 30 分钟文档治理模板

> 目的：稳定执行文档治理机制，持续降低 token 消耗与交接时间。  
> 周期：每周 1 次（建议固定到周三）。  
> 参与角色：流程负责人、质量负责人、圆桌机制负责人（可由同一人兼任）。

## 0) 会前输入（5 分钟准备）

- 时间窗：最近 7 天
- 必备输入：
  - `docs/status/current-status.md`
  - `docs/process-registry.md`
  - `docs/quality/system-functional-matrix.md`
  - `docs/roundtable/consensus.md`
  - `docs/governance/ssot-map.md`
  - `docs/plans/2026-03-05-doc-governance-rollout-plan.md`

## 1) 会议流程（30 分钟）

### A. 快速盘点（5 分钟）

- 本周是否出现“找不到信息 / 不敢用 / 重复冲突”？
- 本周是否出现非 SSOT 文档大段复制正文？
- 本周是否出现 API、状态机、验收口径冲突？

### B. 三项检查（15 分钟）

1. 冲突清单（5 分钟）
   - 新增冲突数：__  
   - 已收敛冲突数：__  
   - 超过 24 小时未收敛项：__

2. 过期清单（5 分钟）
   - 关键文档缺失 `owner` / `last_verified_at`：__  
   - 超过 14 天未复核文档：__

3. 引用合规（5 分钟）
   - 抽查文档数：__  
   - “摘要 + 链接”合规率：__%
   - 违规样例路径：__

### C. 决策与行动（10 分钟）

- 本周优先收敛 Top 3：
  1) <问题 + 负责人 + 截止日期>
  2) <问题 + 负责人 + 截止日期>
  3) <问题 + 负责人 + 截止日期>
- 是否新增/更新 DR（决策记录）：是 / 否
- 是否需要更新 `consensus.md`：是 / 否

## 2) 会后输出（10 分钟内完成）

- 更新治理状态：
  - `docs/plans/2026-03-05-doc-governance-rollout-plan.md`
- 如有决策变更，更新：
  - `docs/roundtable/consensus.md`
  - `docs/decisions/` 下新增或更新 DR 文件
- 如有冲突收敛，回填对应 SSOT 文档：
  - `current-status` / `process-registry` / `system-functional-matrix`

## 3) 会议纪要模板（复制使用）

```markdown
# 文档治理周会纪要（YYYY-MM-DD）

- 主持：<...>
- 参与：<...>
- 时间：30 分钟

## 本周结论
- <结论 1>
- <结论 2>

## 检查结果
- 冲突清单：新增 __ / 收敛 __ / 超时 __
- 过期清单：缺元信息 __ / 超期未复核 __
- 引用合规率：__%

## 行动项（最多 3 条）
1. <负责人> 在 <日期> 前完成 <动作>
2. <负责人> 在 <日期> 前完成 <动作>
3. <负责人> 在 <日期> 前完成 <动作>
```

## 4) 退出条件（判定治理有效）

- 连续 2 周冲突超时项为 0
- 连续 2 周引用合规率 >= 90%
- 连续 2 周交接时间较基线下降 >= 35%
