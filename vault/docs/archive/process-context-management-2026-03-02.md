# Vaultcare OS - 上下文管理文档

> **目的**：解决每次新开对话后，AI 不理解项目进度和需求的问题
> **使用方法**：新对话开始时，先读取本文件，然后读取 `CURRENT_STATUS.md`

---

## 一、问题背景

### 痛点
1. 每次新开对话，AI 从零开始了解项目
2. 重复解释需求和进度，浪费时间
3. AI 给出的建议可能基于过时的理解
4. 开发进度无法有效传承

### 根本原因
- Cursor 对话有 token 限制，无法携带完整上下文
- 项目文档分散在多个文件，AI 不知道读哪个
- 缺少"当前状态"的实时记录

---

## 二、解决方案：三层上下文体系

```
┌─────────────────────────────────────────────────────────┐
│  Layer 1: CONTEXT.md (本文件)                            │
│  - 项目全景图                                            │
│  - 文档索引                                              │
│  - 固定信息（架构、技术栈、业务模式）                     │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Layer 2: CURRENT_STATUS.md (实时状态)                   │
│  - 当前迭代目标                                          │
│  - 已完成/进行中/待办任务                                 │
│  - 最新决策和变更                                        │
│  - 已知问题和 TODO                                       │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Layer 3: 对话记录 (Conversation Log)                    │
│  - 每次对话的摘要记录                                    │
│  - 关键决策的时间戳                                      │
│  - AI 建议的采纳情况                                      │
└─────────────────────────────────────────────────────────┘
```

---

## 三、新对话启动流程

### 标准操作流程（SOP）

**第 1 步：读取上下文索引**
```
请读取：docs/04-context-management.md
```

**第 2 步：读取当前状态**
```
请读取：docs/CURRENT_STATUS.md
```

**第 3 步：确认任务**
```
根据 CURRENT_STATUS.md 中的"进行中"任务，继续工作
或
根据用户的新指令，更新 CURRENT_STATUS.md
```

### 快速启动命令（推荐用户复制）

```
请读取 docs/04-context-management.md 和 docs/CURRENT_STATUS.md，然后告诉我当前项目状态和下一步工作。
```

## 四、文档维护规范

### 4.1 CURRENT_STATUS.md 更新时机

**必须更新的情况：**
1. ✅ 完成一个功能模块
2. ✅ 开始一个新的迭代周期
3. ✅ 做出重大架构决策
4. ✅ 发现并记录重要问题
5. ✅ 用户需求变更

**更新责任人：**
- AI 在完成任务后，主动更新
- 用户在对话结束时，可以要求 AI 总结并更新

### 4.2 对话记录格式

每次对话结束时，AI 应生成如下摘要：

```markdown
## 对话记录：2026-03-02

**时间**：2026-03-02 14:30-15:30
**主题**：PIM 系统功能分析和示例数据生成
**完成工作**：
1. 分析当前系统功能状态
2. 创建 seed_data.py 示例数据脚本
3. 设计上下文管理方案

**关键决策**：
- 确认 WP Webhook 已完成（之前分析有误）
- 优先级调整：拒收流程 > PostgreSQL 迁移

**待办事项**：
- [ ] 用户执行 seed_data.py 生成测试数据
- [ ] 验证 WP Webhook 功能
```

## 七、AI 行为准则

### 对话开始时
1. ✅ 主动读取 `docs/04-context-management.md`
2. ✅ 主动读取 `docs/CURRENT_STATUS.md`
3. ✅ 向用户确认当前任务
4. ❌ 不要基于过时的理解给出建议

### 对话结束时
1. ✅ 生成对话摘要
2. ✅ 更新 `CURRENT_STATUS.md`（如果任务状态变更）
3. ✅ 提醒用户下一步行动

### 遇到不确定时
1. ✅ 承认不确定性
2. ✅ 请求用户澄清
3. ✅ 查阅相关文档
4. ❌ 不要猜测或假设
