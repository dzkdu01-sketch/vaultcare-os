# 问题解决总结 - 2026-01-28

## 📋 本次解决的问题

### 问题1：API Key混淆导致连接失败

**现象**：
- WAHA Dashboard显示"服务器连接失败"
- 错误提示："请确保它在线并在配置中设置正确的API 密钥"

**根本原因**：
- 在WAHA Dashboard的"服务器"配置中，将**Dashboard密码**误填为**API Key**
- API Key和Dashboard密码是两个不同的配置项，用途不同

**解决方案**：
- 明确区分API Key和Dashboard密码
- 在WAHA Dashboard的"服务器"配置中使用正确的API Key：`09972be208234342b1a0999e4426e863`
- Dashboard密码仅用于登录Dashboard界面

### 问题2：API Key不固定

**现象**：
- 容器重启后API Key改变
- 需要频繁更新n8n配置

**根本原因**：
- WAHA容器默认每次启动都会生成新的API Key
- 没有通过环境变量固定API Key

**解决方案**：
- 使用环境变量固定API Key
- 容器启动命令中添加：`-e WAHA_API_KEY=09972be208234342b1a0999e4426e863`
- 确保容器重启后API Key保持不变

## 🎯 关键经验

### 1. 配置项区分的重要性

**API Key** (`09972be208234342b1a0999e4426e863`)：
- 用于API调用认证
- 配置在n8n HTTP Request节点的`X-Api-Key`头部
- 配置在WAHA Dashboard的"服务器"配置中的"API 密钥"字段

**Dashboard密码** (`b4aaddedf6224eb2afe5ef6dfadba381`)：
- 仅用于登录WAHA Dashboard界面
- **不是**API Key，不能用于API调用

### 2. 固定配置的重要性

- 使用环境变量固定关键配置（API Key、密码等）
- 避免容器重启后配置改变导致的问题
- 减少维护成本

### 3. 配置验证

- 通过截图和实际配置界面确认问题
- 不要假设配置正确，需要实际验证
- 区分不同配置项的用途和位置
