# 圆桌纪要：圆桌 v2 与发散-提取-固化机制升级

> 日期：2026-03-04  
> 主题：飞轮进化 — 圆桌优化 + 人机协作整体思路  
> 产出：`docs/roundtable/roundtable-v2-protocol.md`、`consensus.md` 更新、`04-圆桌.md` 更新

---

## 一、讨论背景

用户提出两个问题：

1. **专家召集方式**：希望 AI 先推荐专家，用户确认名单后可增补人员。是否合理？
2. **人机协作整体思路**：采用「发散想法 → 提取想法 → 固化规则」的思路。现有机制是否已有此设计？是否合理？

---

## 二、结论

- 问题 1：**合理**，建议落地为圆桌 v2 的必执行步骤。
- 问题 2：**思路正确**，现有意图-圆桌-飞轮三层已有雏形，但「提取/固化」流程不够标准化，需补充模板和验收门槛。

---

## 三、落地决策

### 3.1 专家召集

| 决策项 | 选择 |
|--------|------|
| 专家召集方式 | AI 推荐 + 用户确认/增补 |
| 未确认时 | 不进入正式讨论 |
| 用户可选动作 | 确认 / 增补：角色名 / 调整：说明 / 跳过（紧急时） |

### 3.2 三阶段流程

| 阶段 | 目标 | 产出 |
|------|------|------|
| 发散想法 | 多视角、无约束 | 专家观点汇总 |
| 提取想法 | 结构化、可决策 | 方案对比表 + 待确认问题（≤5）+ 建议决议 |
| 固化规则 | 写入规范 | consensus.md、意图文档、session 纪要 |

### 3.3 飞轮指标最小集

| 指标 | 用途 |
|------|------|
| 专家组合有效性 | 会后用户反馈，优化专家路由 |
| 追问轮次 | 衡量发散-提取效率 |
| 完成率 | 意图流程完成情况 |
| 返工率 | 提取准确度 |
