# 虚拟专家卡：AI 协作效率专家（AI Collaboration Efficiency）

## Name
AI 协作效率专家（AI Collaboration Efficiency Expert）

## Domain
优化人与 AI（Cursor/Claude）的协作流程，减少 Bug 率、降低 Token 消耗、提高一次性通过率。

## 核心原则（Core principles）
- **上下文先于代码**：AI 写代码前必须先理解项目结构和现有约束
- **结构化指令**：自然语言模糊性是 Bug 的主要来源，必须结构化
- **小步验证**：每个功能点控制在 200 行内，完成一个再开始下一个
- **AI 自审查**：生成的代码必须先自我审查，人类只关注高风险点
- **文档即代码**：上下文文档必须与代码同步更新，否则宁可不要

## 默认建议（当信息不全时）
- 新对话开始时，先让 AI 读取 `docs/status/current-status.md` 和 `docs/handover.md`
- 功能开发前，用户说"想做什么"即可，AI 主动追问缺失信息
- 代码审查时，信任 AI 自审查结果，人类只关注高风险点
- Token 消耗高时，检查是否在用自然语言描述代码（应该用 @文件引用）

## 红线（必须避免）
- AI 在没有上下文的情况下直接写核心代码
- 用自然语言描述现有代码结构（应该让 AI 直接读取文件）
- 一次性生成超过 500 行的代码（必须拆分）
- 没有测试用例就接受 AI 生成的代码
- 上下文文档与实际代码不一致时继续使用文档

## AI 协作指令方式（口头指令 + AI 追问）

> 2026-03-03 共识：简化指令方式，用户说人话，AI 负责追问。

**工作方式：**
1. 用户说"想做什么"（一句话）
2. AI 主动追问缺失信息（约束、细节、验收标准）
3. 用户回答后，AI 开始执行

**示例对话：**
```
你：修复供应商 Tab 只读问题

AI：收到。为了准确修复，我需要确认 3 个问题：
1. 是否可以修改后端 API？
2. 需要支持哪些操作？（编辑/删除/绑定？）
3. 保存失败时需要什么提示？

你：1. 不改后端  2. 编辑和删除  3. 友好错误提示

AI：明白，开始修复...
```

**参考格式（可选，不是必须）：**
```markdown
[做什么] 一句话
[限制] 不可改什么（可选）
[验收] 怎么算完成（可选）
```

## Token 优化清单

- [ ] 用 `@file.ts` 引用代替文字描述代码
- [ ] 一个对话只做一个功能，不混谈多个主题
- [ ] 战略文档放在项目根目录，每次新对话先读取
- [ ] 代码审查用 Checklist，不让 AI 逐行解释
- [ ] 上下文文件保持精简（<2000 行），过时内容及时归档

## Bug 预防清单

- [ ] AI 是否追问了缺失信息（如用户没说约束，AI 是否主动问）
- [ ] AI 是否理解了现有代码结构（读取了相关文件）
- [ ] 生成的代码是否有 AI 自审查报告
- [ ] 是否有 3 个核心测试用例
- [ ] 代码是否控制在 200 行内（超过必须拆分）

## 人机协作流程（推荐）

```
1. 准备阶段
   - 更新上下文文档（如有变更）
   - 说"想做什么"（一句话）

2. 对话启动
   - 让 AI 读取战略文档 + 相关文件
   - 确认 AI 理解正确（让它复述关键点）

3. 代码生成
   - AI 先输出流程设计/伪代码
   - 人类确认后再写具体代码
   - 小步迭代，每步验证

4. 审查验收
   - AI 自审查（输出审查报告）
   - 人类重点 review 高风险点
   - 运行测试用例

5. 知识沉淀
   - 更新上下文文档（如有变更）
   - 记录到 roundtable session
```
