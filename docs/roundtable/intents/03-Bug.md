# 意图：/Bug

**别名：** `/问题` `/报障`

**触发时机：** 报告已知问题

---

## 作用

结构化记录 Bug，自动分配优先级，写入状态文件。

---

## 执行流程

```
1. 追问：现象/复现步骤/期望行为
2. 追问：影响范围（所有用户/部分）
3. 追问：发现时间（新出现/一直有）
4. 自动分配优先级（P0/P1/P2/P3）
5. 记录到 current-status.md
6. 询问："现在开始修复吗？"
```

---

## Bug 优先级定义

| 优先级 | 定义 | 响应时间 |
|--------|------|----------|
| P0 | 阻断性问题，核心流程无法使用 | 立即 |
| P1 | 功能错误，有替代方案 | 24 小时 |
| P2 | 体验问题，不影响使用 | 本周 |
| P3 | 优化建议 | 排期 |

---

## 示例

**用户：** `/Bug 供应商 Tab 不能编辑`

**AI：**
```
收到。为了记录这个 Bug，我需要确认 3 个问题：

1. **现象**：点击编辑按钮没反应，还是有报错？
2. **复现**：每次都能复现吗？需要特定账号吗？
3. **期望**：点击后应该弹出编辑表单？

用户：点击没反应，每次都能复现，应该弹出表单

AI：
---
## Bug 记录

**ID:** Bug-09
**优先级：** P1（功能错误）
**现象：** 供应商 Tab 点击编辑按钮无响应
**复现：** 100%
**期望：** 弹出编辑表单

已记录到 current-status.md

现在开始修复吗？（是/否）
```
