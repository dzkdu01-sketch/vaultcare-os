# 圆桌 v2 开场协议（子项目版）

> 版本：2.0（2026-03-04）  
> 思路：**上下文注入 → 发散想法 → 提取想法 → 固化规则**

---

## 一、开场：专家推荐与确认（强制）

**流程：**

```
用户：/圆桌 [主题]
  ↓
AI：输出「推荐专家名单 + 理由 + 缺失风险」
  ↓
用户：确认 / 增补 / 调整
  ↓
确认后 → 进入「业务上下文注入」再开始发散
```

**推荐名单输出模板：**

```markdown
## 专家推荐

**问题类型：** [效率/质量/流程/架构/混合]

| 序号 | 专家角色 | 必要性（一句话） |
|------|----------|------------------|
| 1 | 流程专家 | 负责流程拆解和边界识别 |
| 2 | AI 协作专家 | 负责人机分工和追问策略 |
| 3 | 执行角色（如审核员） | 负责落地视角和约束条件 |

**若缺某角色可能导致：** [简要说明漏角色的风险]

请确认名单：[确认 / 增补：角色名 / 调整：说明]
```

---

## 二、标准流程：注入 → 发散 → 提取 → 固化

### 阶段 0：业务上下文注入（防瞎出主意）
- 主动读取 `docs/status/current-status.md` / `docs/handover.md` /（如存在）`docs/process-registry.md`
- 若关键信息缺失，必须追问真实流程、痛点、客观约束
- 输出「已拉齐的业务上下文」供用户确认

### 阶段 A：发散（AI 主力）
- AI 必须主动提出 **≥3 个差异化方向**
- 强制引入“红队/极端约束”视角碰撞
- 在提供足够方案前，禁止反问用户“你有什么想法”

### 阶段 B：提取（结构化产出）
必须输出：
- 方案对比表（≥3）
- 待确认问题（≤5，封闭式优先）
- 建议决议（1 句，若可给出）

标准模板：

```markdown
## 方案对比表
| 维度 | 方案 A | 方案 B | 方案 C |
|------|--------|--------|--------|
| 核心思路 | ... | ... | ... |
| 优点 | ... | ... | ... |
| 缺点 | ... | ... | ... |
| 推荐指数 | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ |

## 待确认问题
1. ...
2. ...

## 建议决议
...
```

### 阶段 C：固化（写入共识）
用户明确说“确认/写入共识/进入下一阶段”等后：
1. 写入 `docs/roundtable/consensus.md`
2. 如涉及意图流程，更新 `docs/roundtable/intents/`
3. 记录本场纪要到 `docs/roundtable/sessions/`
