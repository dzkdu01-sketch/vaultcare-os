# AI 协作入口（IDE 通用）

> 本文件是 **任何 IDE / 任何 AI 助手** 的统一入口。  
> Cursor 用户仍会额外受到 `.cursor/rules/collab.mdc` 的强制约束，但**机制正文以 docs 为准**。

---

## 0) 你要怎么开始

### 常用入口
- **新会话**：发 `/启动`
- **开发功能**：发 `/开发 + 你的目标一句话`
- **报 Bug**：发 `/Bug + 现象一句话`
- **复杂问题讨论**：发 `/圆桌 + 主题`

### 强制输出前缀（每条回复开头）
必须以模式标签开头，例如：
- `[人机协作-日常]`
- `[人机协作-启动]`
- `[人机协作-开发]`
- `[人机协作-Bug]`
- `[人机协作-圆桌]`

---

## 1) 新会话必读顺序

1. `docs/status/current-status.md`
2. `docs/handover.md`
3. `docs/roundtable/INTENTS-GUIDE.md`
4. （按需）`docs/flywheel/index.md`

---

## 2) 开发硬门禁（进入 `/开发` 前必须闭环）

1. **唯一业务目标**（1 句话）
2. **明确不做项**（至少 1 条）
3. **验收标准**（3-5 条，至少 1 条页面行为）

> 门禁未闭环：禁止进入编码阶段。

---

## 3) 安全边界（强制）

- **禁止敏感信息入库**：API Key / 密码 / Token 不允许写入 `docs/**`、`.cursor/rules/**`。
- 凭据请放在：`.env` / 部署配置 / 密码管理器（或本地私密文件，不提交 git）。
