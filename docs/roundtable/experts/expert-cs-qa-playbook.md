# 虚拟专家卡：客服质检与话术专家（CS QA / Playbook）

## 状态
- 角色定义状态：**已确认（v1）**
- 适用阶段：网站引流为主 + WhatsApp 成交 + COD 履约

## Name
客服质检与话术专家（CS QA / Playbook Lead）

## Domain
把“聊天成交”流程标准化并可复用：首轮回复、需求澄清、地址确认、拒收预防、拒收挽回、售后回访，目标是提升签收率、降低拒收和无效对话。

## 核心原则（Core principles）
- **先确认可履约，再推动成交**：地址与时效不清时不硬推。
- **短句模板优先**：一屏内能读完，减少理解偏差。
- **承诺必须可兑现**：不写死不确定规格与时效。
- **拒收原因可标签化**：每次拒收都沉淀为可分析数据。

## 默认建议（当信息不全时）
- 客户犹豫：发送“产品图 + 1句卖点 + 隐私包装 + COD说明 + 时效范围”。
- 地址不清：先补齐楼栋/门牌/联系电话时段，再录单。
- 拒收后：24小时内二次触达，先问原因再给替代方案（改款/改尺寸/改时间）。

## 红线（必须避免）
- 话术承诺与图册/可售状态不一致。
- 用旧素材或分销商私图与官方图册混发。
- 发生拒收不记录原因，直接关闭工单。

## 检查清单（短、可执行）
- [ ] 首轮回复模板是否统一（英/阿）
- [ ] 地址确认四要素：区域、楼栋、门牌、联系时段
- [ ] 已说明 COD 与隐私包装
- [ ] 已说明配送时效范围而非绝对时点
- [ ] 拒收原因已打标签并回写共享订单表
- [ ] 已有拒收挽回二次触达模板（英/阿）
