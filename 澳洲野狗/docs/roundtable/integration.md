## 各层职责

### 第一层：意图系统（日常协作）

**作用：** 高效执行已知流程，降低输入成本

**使用场景：**
- 新对话开始：`/启动`
- 开发新功能：`/开发`
- 报告问题：`/Bug`
- 代码审查：`/审查`
- 生成测试：`/测试`
- 更新文档：`/文档`
- 同步进度：`/进度`
- 解释代码：`/解释`
- 排查错误：`/调试`
- 优化代码：`/重构`
- 技术问答：`/提问`
- 项目检查：`/检查`

**特点：**
- 高频使用（每天多次）
- 流程标准化
- AI 主动追问
- 快速执行

---

### 第二层：圆桌头脑风暴（复杂问题）

**作用：** 解决模糊、复杂、多方案的问题

**触发条件：**
| 条件 | 说明 | 示例 |
|------|------|------|
| 意图无法解决 | 问题超出单一意图范围 | "如何优化整个开发流程" |
| 多方案讨论 | 需要对比多个方案 | "选 A 架构还是 B 架构" |
| 跨角色共识 | 需要多专家视角 | "审单流程涉及多方" |
| AI 建议 | AI 识别为复杂问题 | AI 提示"这个问题适合圆桌讨论" |

**流程：**
```
1. 用户发起：/圆桌 [主题]
   或 AI 建议：这个问题适合圆桌讨论

2. AI 识别问题类型 → 召集对应专家

3. 专家讨论 → 输出方案对比表

4. 列出待确认问题（<=5 个）

5. 用户确认 → 写入 consensus.md
```

**特点：**
- 低频使用（每周 0-2 次）
- 多专家参与
- 输出方案对比
- 形成共识决策

---

### 第三层：飞轮进化（持续改进）

**作用：** 让系统越用越好

**数据收集：**
| 数据类型 | 收集方式 | 频率 |
|----------|----------|------|
| 意图使用统计 | 自动记录 | 每日 |
| 意图完成率 | 自动记录 | 每日 |
| 圆桌决策执行率 | 手动追踪 | 每周 |
| Bug 复发率 | 自动记录 | 每周 |
| 用户满意度 | 手动评分 | 每次协作后 |

**优化周期：**
| 周期 | 内容 | 产出 |
|------|------|------|
| 每日 | 数据收集 + 问题记录 | 使用统计 |
| 每周 | 小优化（Checklist/模板微调） | 优化建议 Top 3 |
| 每月 | 大优化（新意图/流程重构/专家卡） | 进化报告 |

**决策流程：**
```
1. AI 分析数据 → 生成优化建议

2. 小优化（Checklist/模板）：
   - AI 建议 → 用户确认 → 直接更新

3. 大优化（新意图/流程重构）：
   - AI 建议 → /圆桌 讨论 → 用户拍板 → 更新
```

**特点：**
- 自动化程度高
- 分层更新（周/月）
- 数据驱动决策

## 三层配合示例

### 示例 1：日常开发 + 问题升级

```
用户：/启动
AI：（读取上下文，输出状态）

用户：/开发 供应商编辑功能
AI：（追问约束和验收，生成代码）

用户：/审查
AI：（输出审查报告，发现高风险问题）

用户：/调试 这个风险怎么解决
AI：（分析原因，给出建议）

用户：（不满意，问题复杂）
AI：这个问题涉及架构设计，建议启动圆桌讨论。

用户：/圆桌 供应商编辑的架构设计
AI：（召集专家，输出方案对比）

用户：确认方案 B
AI：（写入 consensus.md）

用户：/开发 按方案 B 重新实现
AI：（执行）

用户：/进度
AI：（记录今日进展，包括圆桌决策）
```

### 示例 2：飞轮进化循环

```
每周五：/总结 本周进展
AI：（生成本周总结）

## 本周意图使用统计
- /开发：5 次
- /Bug：3 次
- /圆桌：1 次
- /调试：4 次

## 高频问题
- 供应商模块问题 3 次

## 建议
- 下周安排供应商模块重构

用户：接受建议
AI：（更新下周计划）

每月：/圆桌 月度流程优化
AI：（召集专家，分析本月数据）

## 发现
- /调试 意图完成率低（40%）
- 原因：追问步骤太多

## 决策
- 简化 /调试 流程

（决策写入 consensus.md）
```

## 意图与圆桌的配合

### 意图 → 圆桌（升级）

```
用户：/调试 XXX
AI：（分析问题，发现涉及多模块）
AI：这个问题涉及多个模块的协作，建议启动圆桌讨论。
是否现在启动？（是/否）

用户：是
AI：好的，启动圆桌讨论...
```

### 圆桌 → 意图（执行）

```
圆桌决策：采用方案 B，重构供应商模块

用户：/开发 按方案 B 重构供应商模块
AI：收到。根据圆桌决策，方案 B 的核心是...
确认开始执行吗？（是/否）
```

## 飞轮与意图的配合

### 意图优化流程

```
1. 飞轮数据收集：
   - /调试 意图完成率 40%（低于平均）

2. AI 分析原因：
   - 追问步骤太多（平均 5 步）
   - 用户容易中断

3. AI 生成优化建议：
   - 简化为 3 步
   - 合并相关问题

4. 每周优化会议：
   - AI 提交建议
   - 用户确认
   - 更新 /调试 意图文档

5. 验证效果：
   - 下周追踪完成率是否提升
```

## 飞轮与圆桌的配合

### 圆桌决策追踪

```
1. 圆桌决策记录：
   - 决策内容
   - 执行负责人
   - 预计完成时间

2. 每周追踪：
   - 决策是否执行
   - 执行效果如何
   - 是否需要调整

3. 每月回顾：
   - 决策执行率统计
   - 成功/失败案例分析
   - 改进建议
```

## 快速参考

### 什么时候用什么？

| 场景 | 使用机制 |
|------|----------|
| 日常开发 | 意图（/开发 /审查 /测试） |
| 遇到问题 | 先意图（/调试），解决不了再圆桌 |
| 方案分歧 | 直接圆桌 |
| 定期回顾 | 飞轮（/总结） |

### 决策权分配

| 变更类型 | 决策者 |
|----------|--------|
| 小优化（Checklist） | AI 建议 + 用户确认 |
| 中优化（意图流程） | 圆桌讨论 + 用户拍板 |
| 大优化（新意图/专家卡） | 圆桌讨论 + 用户拍板 |
