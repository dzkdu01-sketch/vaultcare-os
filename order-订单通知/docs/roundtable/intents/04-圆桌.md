# 意图：/圆桌

**触发方式：** `/圆桌` 或 `圆桌、`、`讨论、`（后接主题）

**别名：** `/讨论` `/brainstorm`

**触发时机：** 复杂问题需要专家讨论

**协议版本：** v2（详见 `docs/roundtable/roundtable-v2-protocol.md`）

---

## 作用

召集虚拟专家，输出方案对比，辅助决策。采用 **上下文注入 → 发散 → 提取 → 固化** 四阶段流程。

---

## 执行流程（v2）

```
1. 识别问题类型（效率/质量/流程/架构）
2. 输出推荐专家名单 + 理由 + 缺失风险 → 等待用户确认/增补
3. 确认后：业务上下文注入（读取文档/追问痛点约束）
4. 发散：AI 必须主动输出 ≥3 个差异化方案方向，并引入红队碰撞
5. 提取：输出方案对比表 + 待确认问题（≤5）+ 建议决议
6. 用户确认后：固化规则，写入 `docs/roundtable/consensus.md`
```

---

## 注意事项（强制）

- 圆桌模式下 **不主动退出**：只有用户明确说“结束/进入下一阶段/切换意图”才退出
- 必须产出：方案对比表 ≥3、待确认问题 ≤5
