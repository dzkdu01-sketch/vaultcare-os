# 意图：/调试

**别名：** `/排查` `/debug`

**触发时机：** 排查错误原因

---

## 作用

系统化定位问题根因，给出排查步骤和修复建议。

---

## 执行流程

```
1. 询问错误现象（报错/无响应/数据不对）
2. 追问错误信息/截图
3. 定位可能原因（按概率排序）
4. 给出排查步骤
5. 确认后执行修复
```

---

## 调试框架

### 现象分类
- 报错类：有明确错误信息
- 无响应类：点击/操作无反应
- 数据类：显示数据不对

### 可能原因（按概率）
1. 事件未绑定
2. 事件处理函数报错
3. 状态未更新
4. 数据源问题
5. 网络问题

---

## 示例

**用户：** `/调试 供应商 Tab 点击编辑没反应`

**AI：**
```
---
## 调试分析

**可能原因（按概率排序）：**

1. **事件未绑定**（30%）
   - 按钮缺少 onClick 处理器

2. **事件处理函数报错**（25%）
   - handleEdit 函数内部错误

3. **状态未更新**（20%）
   - showEditModal 状态未设置

4. **数据源问题**（15%）
   - supplier 数据为空

5. **网络问题**（10%）
   - API 加载失败

---
**排查步骤：**

1. 打开浏览器控制台
2. 点击编辑按钮
3. 查看是否有报错

请告诉我控制台有什么输出？
```
