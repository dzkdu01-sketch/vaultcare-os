> 日期：2026-03-09
> 状态：需求确认版（v2）
> 作用：定义老板、运营审单、分销商三个角色对 8 个页面的页面级、字段级、操作级权限，以及留痕范围和草稿责任边界，作为 Gate 3（数据门）的核心交付物
> 上位文档：`05-页面字段字典.md`

---

## 1. 角色定义

### 1.1 老板

职责边界：

- 全局业务监控，查看所有模块的数据和状态
- 关键决策审批：冻结范围、确认成功指标、确认商品可分销的最小门槛
- 不参与日常操作，不直接编辑商品、订单或分销数据
- 可查看所有成本、供应商、分销商信息

### 1.2 运营审单

职责边界：

- 商品模块的主要操作者：负责商品入池、建档、AI 辅助处理、素材处理、推进商品到可分销状态
- 分销模块的主要操作者：负责分销商和站点管理、选品发布、发布异常处理
- 订单模块的主要操作者：负责手动录入订单、审核订单、拆单、交接、异常处理、拒收任务处理
- 可查看所有成本、供应商、分销商信息
- 可查看所有分销商的数据

### 1.3 分销商

职责边界：

- 只能查看与自己关联的分销和订单数据，不能查看其他分销商的数据
- 不能操作商品模块（商品工作台首页、商品详情页不可见）
- 不能查看商品成本价、供应商信息
- 可查看自己站点的选品发布状态和订单状态
- 不能执行审核、拆单、交接等运营操作

| 操作前状态 | string | 状态变更前的值（状态变更类动作） |
| 操作后状态 | string | 状态变更后的值（状态变更类动作） |
| 操作人 | string | 执行操作的用户 |
| 操作人角色 | enum | 老板 / 运营审单 / 分销商 |
| 操作时间 | datetime | |
| 操作备注 | text | 驳回原因、关闭原因等附加信息 |
| 关联对象 | string | 可选，如拆单时关联父订单号 |

## 6. 草稿责任边界

### 6.1 商品草稿

| 责任项 | 责任方 | 说明 |
|---|---|---|
| 草稿创建 | 运营审单 | 通过"新建商品"或"导入资料"创建 |
| 草稿内容填写 | 运营审单 | 负责填写来源、标题、供应商、货权类型等基础信息 |
| 草稿质量 | 运营审单（创建者） | 创建者对草稿内容的准确性负责 |
| 草稿推进 | 运营审单 | 负责推进商品通过各门禁直到可分销 |
| 草稿长期未处理 | 运营审单（创建者） | 草稿超过约定时间未推进，由创建者负责跟进或关闭 |

### 6.2 订单草稿（仅非网站渠道手动录入）

| 责任项 | 责任方 | 说明 |
|---|---|---|
| 草稿创建 | 运营审单 | 通过"手动录入订单"创建 |
| 草稿信息准确性 | 运营审单（创建者） | 创建者对客户信息、商品信息、金额的准确性负责 |
| 客人确认 | 运营审单（创建者） | 创建者负责确认客人意向后勾选"客人已确认" |
| 提交待审核 | 运营审单（创建者） | 创建者负责在信息完整后提交待审核 |
| 审核责任 | 运营审单（审核人） | 审核人对审核结果负责，审核人不应与创建者为同一人（建议，非强制） |
| 驳回后补充 | 运营审单（创建者） | 被驳回的订单由创建者负责补充信息后重新提交 |

### 6.3 WordPress 自动接入订单

| 责任项 | 责任方 | 说明 |
|---|---|---|
| 数据接入准确性 | 系统 | webhook 数据自动解析，系统对解析结果负责 |
| 数据异常处理 | 运营审单 | 自动接入数据有异常时，由运营审单负责标记异常并处理 |
| 审核责任 | 运营审单（审核人） | 审核人对审核结果负责 |
