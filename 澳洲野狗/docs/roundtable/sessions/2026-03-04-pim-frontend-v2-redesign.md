# 圆桌纪要：PIM 前端页面重构（V2 蓝图）

- **日期**：2026-03-04
- **主题**：结合业务和流程，PIM 前端页面的重构设计与核心痛点解决
- **触发痛点**：“操作效率太低，走不通流程，不会用”
- **参与专家**：PIM 业务专家、前端架构与 UX 专家、商品管理员/审核员、分销商代表、系统管理员、老板、PM

---

## 一、 核心发散与决策路径

### 1. 基础架构重组（解决“不会用”）
- **痛点**：全状态混杂，页面跳转多，学习成本高。
- **决策**：改为**全尺寸页面**，采用**四大 Tab 驱动（我的草稿 / 待审核 / 商品库 / 异常池）**。取消详情页全页跳转，改为点击列表**右侧滑出宽抽屉**进行快编快审。支持抽屉内“审核并下一条”连贯操作。

### 2. AI 辅助上新融合（解决“录入繁琐”）
- **痛点**：当前 AI 录入是独立的多步跳转页，与主流程割裂。
- **决策**：将 AI 录入做成**独立抽屉**，支持拖拽传图。AI 完成翻译（英/阿）与结构化卖点提炼后，表单分为**【AI提取区】**与**【人工必填区】**（价格/SKU）。高危字段（品类/标签）强制需要员工点击确认方可保存。

### 3. 录入规范的前端固化（防呆防错）
- **痛点**：规范靠人背，短描述写成作文，驳回率高。
- **决策**：标题框悬浮公式提示并截断超长字符；**短描述废弃长文本框，改为条目列表组件**；抽屉内集成“合规性检测”与“一键带原因驳回”。

### 4. 基础字典的闭环管理（防数据污染）
- **痛点**：缺少分类、标签、品牌等维护入口，AI 容易产生“脏词”。
- **决策**：新增高级管理员专属的**字典维护模块（分类树、标签、品牌等）**；禁止物理删除。AI 触发脏词时，前端强拦截并标红，倒逼人工选择标准词。

### 5. 高阶潜在需求（预留扩展）
- **深挖痛点**：未来录入效率与多站定价可能再次引发重构。
- **决策**：UI 架构上预留**【URL爬虫抓取】**和**【站点定价矩阵】**的空间，暂不实施全自动清洗和动销回流面板。

---

## 二、 门禁与废弃说明

- **撤销老板特权**：原 PIM-F13 中的“老板紧急放行通道”在此次重构中被否决。任何人必须遵守标准门禁：`草稿 -> 待审 -> 可发布`。
- **分销商只读约束**：分销商打开商品抽屉时，内部运营/成本等敏感字段强制只读，仅保留选品推送权限。
