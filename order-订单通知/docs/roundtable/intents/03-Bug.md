# 意图：/Bug

**别名：** `/问题` `/报障`

**触发时机：** 报告已知问题

---

## 作用

结构化记录 Bug，分配优先级，写入状态文件。

---

## 执行流程

```
1. 追问：现象/复现步骤/期望行为
2. 追问：影响范围（所有用户/部分）
3. 追问：发现时间（新出现/一直有）
4. 分配优先级（P0/P1/P2/P3）
5. 记录到 docs/status/current-status.md
6. 询问："现在开始修复吗？"
```

---

## Bug 优先级定义

| 优先级 | 定义 | 响应时间 |
|--------|------|----------|
| P0 | 阻断性问题，核心流程无法使用 | 立即 |
| P1 | 功能错误，有替代方案 | 24 小时 |
| P2 | 体验问题，不影响使用 | 本周 |
| P3 | 优化建议 | 排期 |
