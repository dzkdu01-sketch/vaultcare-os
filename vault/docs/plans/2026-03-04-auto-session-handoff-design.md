# 自动切会话与续航机制设计（v1，激进型）

> 日期：2026-03-04  
> 适用范围：Vault 人机协作会话（默认日常/开发/Bug/圆桌等模式）  
> 决策来源：用户已确认采用「A 激进型」策略

---

## 1. 目标与边界

### 1.1 目标

- 在上下文膨胀、目标漂移或重复澄清时，尽早触发“新会话续航”，降低失焦和返工率。
- 通过标准化“续航包”让新会话可直接进入执行状态，而不是重复回顾历史。
- 将触发行为纳入飞轮指标，持续校准阈值灵敏度。

### 1.2 本版边界（v1）

- v1 不做“平台级自动新建会话”，只做“自动建议切换 + 自动生成续航包”。
- v1 不接入外部消息提醒通道，不做跨工具通知。
- v1 先在文本流程中落地，后续再考虑脚本化或插件化执行。

---

## 2. 方案选择与理由

### 2.1 已确认方案

- 采用 **方案 A：硬阈值直切（激进型）**。
- 触发策略：命中任一阈值即触发“建议新会话”。

### 2.2 采用理由

- 当前任务类型以多流程协作与规则固化为主，稳定性优先于连续聊天体验。
- 激进型能更早阻断上下文污染，减少“重复追问-重复澄清”的低效回路。

---

## 3. 触发规则（v1）

> 命中任一条件即触发（OR 逻辑）

1. **轮次阈值**：有效对话轮次 `>= 28`。  
2. **待确认积压**：待确认问题累计 `>= 6`。  
3. **重复澄清**：同一阻塞问题连续重复澄清 `>= 2` 次。  
4. **目标冲突**：目标/边界发生二次改写且与既有决策冲突。  
5. **决策回滚迹象**：已确认决策在当前会话被反向重提并影响执行。

---

## 4. 触发后的自动动作

### 4.1 动作清单

触发后，AI 必须按顺序输出：

1. 一句话说明触发原因（命中哪条规则）。  
2. 生成标准化“续航包（5段）”。  
3. 给出统一续航指令：  
   `请新开会话并发送：/启动 + 以下续航包`  
4. 标记当前会话状态：`已建议切换（原因：xxx）`。

### 4.2 防抖与冷却

- **冷却窗口**：8 轮内最多触发 1 次建议。  
- **延后触发白名单阶段**：正在执行命令、跑测试、落库写文档时，先完成当前动作再触发建议。

---

## 5. 续航包标准模板（可直接复制）

```markdown
【续航包 v1】
1) 当前目标：
- （一句话，描述本轮必须达成的结果）

2) 边界 / 不做：
- （最多 3 条，明确本轮不做什么）

3) 已完成（含证据）：
- （事项 + 证据路径/结果，例如 docs/...、测试通过条目）

4) 未完成（最多 3 条）：
- [P0] ...
- [P1] ...
- [P1] ...

5) 下一步第一动作（单句可执行）：
- （例如：先补齐 xxx 文档，再执行 yyy 回归）
```

---

## 6. 意图体系集成点

### 6.1 与 `/启动`、`/继续` 的衔接

- 新会话推荐入口：`/启动 + 续航包`（首选）。  
- 同会话恢复入口：`/继续`（当用户明确不切会话时使用）。

### 6.2 与圆桌 v2 的衔接

- 圆桌模式不主动退出原则保持不变。  
- 若在圆桌中命中触发阈值，仅可“建议切换并生成续航包”，不得擅自切离圆桌阶段。  
- 只有用户明确确认后，才执行会话迁移。

---

## 7. 飞轮指标回流（v1 最小集）

每次触发记录以下字段（用于周度复盘）：

- 触发次数  
- 触发原因分布（轮次/积压/重复/冲突/回滚）  
- 用户采纳率（采纳切换次数 / 建议次数）  
- 采纳后完成率（切换后完成任务数 / 切换总数）

### 7.1 调参建议（周度）

- 若建议过多且采纳率低：提高阈值（如 28 -> 32）。  
- 若建议过少但返工率高：降低阈值（如 28 -> 24）并缩短冷却窗口。  
- 若重复澄清频发：优先收紧“重复澄清”触发条件。

---

## 8. 验收标准（DoD）

- 文档内存在可执行触发条件，且为明确数值阈值。  
- 文档提供可直接复用的续航包模板。  
- 明确防抖规则与白名单延后规则。  
- 明确与意图系统、圆桌 v2 的边界关系。  
- 明确飞轮回流指标与调参方向。

## 10. v1.2 执行策略补充（2026-03-04）

### 10.1 自动直推（默认）

- 当用户明确“继续/直接做/不需要确认”后，AI 进入自动直推模式。
- 自动直推模式下，默认连续执行，不做逐步征求确认。
- 仅在“阻塞/高风险/冲突”三类场景打断。

### 10.2 三联动收口（每轮必做）

每轮动作完成后，必须自动执行：

1. 状态证据回填：`docs/status/current-status.md`
2. 自动交接快照：`docs/handover/auto/session-*.md` + `docs/handover.md`
3. 飞轮回流：`docs/flywheel/weekly-report-*.md` + `docs/flywheel/metrics-YYYY-MM.json`

### 10.3 上下文高压自动续航

- 触发条件：会话接近上下文上限或出现明显上下文膨胀。
- 必做动作：
  - 输出续航包 v1（目标/边界/已完成证据/未完成<=3/下一步第一动作）
  - 写入 auto handover 快照
  - 输出新会话入口：`/启动 + 续航包`
