## 一、生意层 (Business Level)

### 1.1 商业模式定位

**你的角色：** 老板 / 操盘手

你身处北京，远程经营一个面向阿联酋市场的成人用品独立站矩阵。你是这盘生意的唯一所有者和决策者。你负责：
- 对接并管理阿联酋本地货盘（供应链核心）
- 广告投放与流量获取（你自己也在投放）
- 为"分销商"（亲友团）提供商品资料、建站技术支持（斗篷等）
- 集中处理订单审核与推送
- 完成资金结算

**"分销商"的角色：亲友团，非商业合作伙伴**

当前确认基线为 **1 个分销商**（可扩展到多人）。该分销商承担：
- 广告费用（自己购买FB账号、自己充值投放）
- 客户沟通（各自运营WhatsApp）
- 网站运营（各自的WordPress站点）

**收入模式（已最终确认）：**
- **分销商收入模式：** 分销商通过销售赚取差价。销售价 300 AED，减去 150 AED 成本（包含货品成本、派送费、拒签费），分销商每单收入 150 AED。
- **老板收入模式：** 老板通过自营站点投放广告带来的成交订单赚取进销差价，同时从分销商支付的成本中获得利润（成本价与供应商采购价之间的差价）。

### 1.4 现金流模型

**资金流向（COD模式）：**
```
客人付280 AED给派送员
        ↓
货盘收到280 AED
        ↓
货盘扣除：货品成本 + 派送费(50) + 操作费(5, QR) + 拒收退回费(10, 如有)
        ↓
剩余部分按结算周期返还给你（所有利润归你，不分给分销商）
        ↓
你通过货盘老板 -> 钱庄 -> 微信转账 收到人民币
```

**结算周期：**
- QR：每2周一次（**目前已超1个月未结，欠款约30,000 AED**）
- VIP：每周一次（正常运作）

**停摆期固定支出（每月）：**
| 项目 | 费用 |
|------|------|
| 审单员工资 | 待确认 |
| QR仓储费 | 250 AED |
| 网站服务器/域名 | 200 RMB/月 |
| IP环境费 | 50 RMB/账号环境/月 |
| 斗篷 | 无（一次性买断） |

**广告停摆率：约50%的时间无法正常投放**
- 原因A：FB账号被封（关联封禁导致集体阵亡）
- 原因B：货盘出问题（QR被查、暂停服务等）
- 停摆期间分销商（亲友）去做其他工作
- 固定成本照付，无收入

## 二、业务层 (Operations Level)

### 2.1 订单全链路流程（已确认）

```
客户看到FB广告
    ↓
【网站引流模式】点击进入网站（斗篷判定：真实用户→A站/审核机器人→B站）
【消息互动模式】直接跳转WhatsApp对话（计划中）
    ↓
客户浏览商品 / 在WhatsApp里咨询
    ↓
通过WhatsApp确认订单（商品、地址、价格）
    ↓
将订单填入共享表格
    ↓
审单员审核（肉眼比对商品→确认货盘归属→确认库存→计算利润→决定是否拆单）
    ↓
审单员将订单推送至QR/VIP各自的表格
    ↓
货盘安排派送员出发（摩托车COD）
    ↓
派送（3-7天，偏远地区一周去一次）
    ↓
签收（70%）/ 拒收（30%，退回费10 AED）
    ↓
结算（VIP每周 / QR每两周，目前QR停摆）
    ↓
货盘老板 -> 钱庄 -> 微信转账人民币
```

## 三、Facebook广告安全评估 (FB Safety Assessment)

### 3.1 投放基础设施隔离度（已修正）

**分销商之间的隔离：**
| 基础设施 | 状态 | 风险评级 |
|---------|------|---------|
| Facebook广告账号 | **隔离** | 低 |
| Business Manager (BM) | **隔离** | 低 |
| Facebook像素 (Pixel) | **隔离** | 低 |
| 投放使用的IP/代理 | **隔离** | 低 |
| 广告素材 | **隔离** | 低 |
| 投放域名（落地页） | **隔离**（各自独立域名） | 低 |
| 白页域名（B站） | **共用** `marmarland.com` | **极高** |
| 斗篷服务/服务器 | **共用** | **高** |
| 广告支付方式（信用卡） | **共用** | **极高** |
| WordPress主机/服务器 | **共用** | **高** |

**你自己的3套账号之间的隔离：**
| 基础设施 | 状态 | 风险评级 |
|---------|------|---------|
| 投放域名（落地页） | **共用**（3个账号投同一个域名） | **极高** |
| 其他维度 | 同上 | 同上 |

**结论：**
- 分销商之间在"上层"（账号、BM、像素、IP、素材、**域名**）的隔离做得不错。
- 但在"下层"（白页、支付、服务器）的共用，形成了3条强关联链路。
- **你自己的3套账号共用一个域名投放，是最直接的关联点，几乎必然会被FB检测到。**

### 3.2 已识别的关键漏洞（5个）

**漏洞1：白页 `marmarland.com` 全员共用（极高风险）**
- 所有人的斗篷跳转都指向同一个白页
- 一个人被查，FB通过白页域名串联所有人
- **用户已要求指导部署独立白页**

**漏洞2：VPN/数据中心拦截未开启（极高风险）**
- FB审核爬虫通过数据中心IP访问时，斗篷未拦截
- 审核机器人可能直接看到了A站（成人用品页面）
- **用户已确认立即开启**

**漏洞3：广告支付方式全员共用（极高风险）**
- 所有人使用同一张信用卡支付广告费
- 这是FB最容易检测的关联信号
- **需要每人使用独立支付方式**

**漏洞4：你自己的3套账号共用域名（极高风险）**
- 你自己的3个广告账号都投向同一个域名
- 一封全封，毫无悬念
- **需要为每套账号准备独立域名**

**漏洞5：用户举报（中等风险）**
- 广告被用户举报过
- 举报 + 斗篷漏洞（VPN未拦截）= 账号被封的组合拳

## 四、2026年战略方向：消息互动广告

### 4.1 为什么要转向消息互动？

当前"网站引流"模式面临的核心矛盾：
1. **CAC过高：** 100 RMB/单的获客成本，吃掉了大量利润。
2. **斗篷依赖极重：** 必须有落地页、斗篷、白页的完整配合，任何一环出问题都会导致封号。
3. **封号频繁：** 50%时间停摆，极不稳定。

消息互动广告的潜在优势：
1. **CAC大幅降低：** 行业参考数据 50 RMB/天可成交3单，即~17 RMB/单。
2. **减少对斗篷的依赖：** 广告直接引导用户到WhatsApp对话，无需落地页，FB审核重点转移到广告素材本身。
3. **可能降低封号概率：** 没有落地页审核环节，减少了一个被检测的维度。

### 4.2 消息互动模式的新挑战

1. **客服人力瓶颈：** 从每天10个对话暴增到100个对话，其中大量是无效沟通（好奇、随便问的），需要大量人工筛选真实客户。现有2人团队无法承受。
2. **客单价下降：** 消息互动吸引的客户可能更冲动、预算更低，平均客单价预计下降（具体降多少待确认）。
3. **签收率可能更低：** 冲动下单的客户，拒收概率可能更高。
4. **SKU展示受限：** 没有网站作为商品展示载体，客户只能在WhatsApp里看图片。如何高效展示230+个SKU？

### 4.3 消息互动模式的待确认问题

1. 行业伙伴的"50 RMB/天成交3单"数据，他们的品类是什么？是同样的成人用品，还是其他品类？
2. 消息互动模式下，广告素材能否延续现有的C类（暗示性内容），还是需要更保守？
3. 如果每天100个对话，你打算用什么方式处理？人工？还是考虑用WhatsApp Business API + 自动回复/聊天机器人？
4. 网站引流和消息互动，你打算双线并行，还是逐步替代？
