## 什么是飞轮进化？

**飞轮进化**是让整个人机协作体系**越用越好**的持续改进机制。

```
┌─────────────────────────────────────────────────────┐
│                  飞轮进化循环                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│   数据收集  →  数据分析  →  优化执行  →  验证      │
│      ↑                                  │          │
│      └──────────────────────────────────┘          │
│                                                     │
└─────────────────────────────────────────────────────┘
```

## 数据收集

### 收集内容

| 数据类型 | 收集方式 | 频率 |
|----------|----------|------|
| **意图使用统计** | 自动记录 | 每日 |
| - 各意图使用次数 | | |
| - 意图完成率 | | |
| - 平均执行时长 | | |
| **圆桌决策追踪** | 手动追踪 | 每周 |
| - 决策数量 | | |
| - 决策执行率 | | |
| - 决策效果评分 | | |
| **Bug 复发率** | 自动记录 | 每周 |
| - 同一问题重复出现次数 | | |
| - Bug 修复成功率 | | |
| **用户满意度** | 手动评分（1-5） | 每次协作后 |

## 数据分析

### 每周分析报告

**AI 自动生成：**

```markdown
## 第 X 周 分析报告

### 高频意图 Top 5
1. /开发 - 15 次
2. /审查 - 10 次
3. /Bug - 8 次
4. /调试 - 5 次
5. /进度 - 5 次

### 低频意图 Top 3（考虑删除/合并）
1. /部署 - 0 次
2. /检查 - 1 次
3. /总结 - 1 次

### 圆桌决策执行率
- 本周决策：3 个
- 已执行：2 个（67%）
- 待执行：1 个

### 优化建议 Top 3
1. 简化 /调试 流程（完成率低，仅 40%）
2. 增加 /部署 意图的使用培训
3. /检查 意图与 /审查 合并
```

### 分析维度

| 维度 | 指标 | 阈值 |
|------|------|------|
| 使用频率 | 周使用次数 | <2 次考虑删除 |
| 完成率 | 完成流程的用户比例 | <50% 需要优化 |
| 满意度 | 用户评分 | <3.5 需要改进 |
| 执行率 | 圆桌决策执行比例 | <70% 需要跟进 |

## 优化执行

### 分层更新策略

| 层级 | 频率 | 内容 | 决策流程 |
|------|------|------|----------|
| **小优化** | 每周 | Checklist/模板微调 | AI 建议 + 用户确认 |
| **中优化** | 每月 | 意图流程优化 | 圆桌讨论 + 用户拍板 |
| **大优化** | 每月 | 新意图/流程重构/专家卡 | 圆桌讨论 + 用户拍板 |

### 优化产出类型

| 产出类型 | 说明 | 频率 | 示例 |
|----------|------|------|------|
| 新意图 | 新增意图词 | 每月 | 新增 `/通知` 意图 |
| 意图优化 | 修改现有意图流程 | 每周 | 简化 `/调试` 追问步骤 |
| 流程优化 | 修改协作流程 | 每周 | 修改代码审查 Checklist |
| 专家卡更新 | 新增/修改虚拟专家 | 每月 | 新增"安全专家"角色 |
| 文档更新 | 更新现有文档 | 每周 | 更新 `INTENTS-GUIDE.md` |
| Checklist 更新 | 新增/修改检查清单 | 每周 | 新增"上线前 Checklist" |
| 模板更新 | 更新各类模板 | 每周 | 更新 Bug 报告模板 |
| 共识决策 | 新的协作规则 | 每月 | "所有 API 变更必须写文档" |
| 工具脚本 | 自动化脚本 | 每月 | 自动生成文档更新建议 |
| 培训材料 | 新人上手文档 | 每月 | "意图系统 5 分钟上手" |

## 验证闭环

### 优化后追踪

```
优化执行 → 追踪效果 → 下一轮数据收集
```

**追踪指标：**

| 优化类型 | 追踪指标 | 成功标准 |
|----------|----------|----------|
| 意图优化 | 完成率提升 | +20% |
| 流程优化 | 执行时间缩短 | -30% |
| Checklist 更新 | 遗漏率降低 | -50% |
| 培训材料 | 新人上手时间 | -50% |

### 验证报告模板

```markdown
## 优化验证报告

### 优化内容
- 简化 /调试 流程（5 步 → 3 步）

### 优化前（第 1 周）
- 使用次数：10 次
- 完成率：40%
- 满意度：3.2/5

### 优化后（第 2 周）
- 使用次数：12 次
- 完成率：75%
- 满意度：4.1/5

### 结论
✅ 优化成功，继续观察

### 后续行动
- 将简化流程应用到其他意图
```

## 飞轮周期

### 每日例行

**时间：** 每天结束时自动执行

**内容：**
```
1. 收集当日使用数据
2. 记录问题和建议
3. 更新进度文件（/进度）
```

**输出：** 每日使用统计

---

### 每周回顾

**时间：** 每周五

**流程：**
```
用户：/总结 本周进展
AI：（生成周分析报告）

1. 意图使用统计
2. 圆桌决策追踪
3. 优化建议 Top 3

用户：确认优化建议
AI：（执行小优化，更新文档）
```

**输出：** 周分析报告 + 优化执行清单

---

### 每月进化

**时间：** 每月最后一个工作日

**流程：**
```
用户：/圆桌 月度流程优化
AI：（召集专家，分析本月数据）

1. 本月数据回顾
2. 大优化讨论（新意图/流程重构）
3. 下月计划

用户：确认方案
AI：（写入 consensus.md，更新专家卡）
```

**输出：** 月度进化报告 + 共识决策

## 飞轮与意图的配合

### 意图使用数据驱动优化

```
数据收集：/调试 完成率 40%
   ↓
数据分析：追问步骤太多（平均 5 步）
   ↓
优化建议：简化为 3 步
   ↓
每周确认：用户同意
   ↓
执行优化：更新 /调试 意图文档
   ↓
验证效果：下周完成率提升到 75%
```
