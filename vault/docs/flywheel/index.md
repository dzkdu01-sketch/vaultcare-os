## 2) 指标口径（最小集）

| 指标 | 来源 | 用途 |
|------|------|------|
| 专家组合有效性 | 圆桌 session 末尾飞轮备注 | 优化专家路由 |
| 追问轮次 | sessions 对话统计 | 评估提问效率 |
| 完成率 | 意图完成数 / 启动数 | 衡量流程可执行性 |
| 返工率 | 用户“否定/重开”次数占比 | 衡量提取准确性 |

## 2.1 人机协作质量补充指标（2026-03-04）

| 指标 | 来源 | 用途 |
|------|------|------|
| 业务问题占比 | session 提问记录 | 检查 AI 是否坚持“业务优先提问” |
| 单轮追问数 | session 提问记录 | 控制提问负担，目标每轮 ≤ 3 个阻断问题 |
| 预填采用率 | session 决策记录 | 衡量“AI 预填 80% + 用户确认”是否落地 |
| 门禁拦截次数 | `/开发` 前拒绝记录 | 评估“先闭环后开发”执行强度 |
| 拦截后一次通过率 | 拦截后任务结果 | 评估门禁质量（防误拦/防漏拦） |

## 3) 更新节奏

- 每场圆桌结束：更新 session 的“本场飞轮备注”
- 每周：更新周报文件并在本索引补充链接
- 每月：更新 `metrics-YYYY-MM.json` 并回看趋势

### 每周 5 分钟复盘（新增）

- 回看本周门禁拦截 Top 3 场景
- 确认哪些追问可以沉淀为“默认预填项”
- 确认是否有“过早讨论 UI/过早进入开发”的复发
- 将通过复盘的规则回写到 `INTENTS-GUIDE.md` 与 Checklist
