**关键交互**：

| 操作 | 触发 | 结果 | 引用 |
|------|------|------|------|
| 点击订单行 | 单击行任意位置 | 跳转页面 7（订单详情） | — |
| 手动录入 | 点击 [+ 手动录入订单] | 弹出录入表单 → 创建草稿 [状态机 T-O1] | [API §4.1] |
| 状态标签页切换 | 点击标签 | 列表按状态筛选刷新 | — |
| WordPress 自动接入 | 站点推送 webhook | 自动创建订单进入 pending_review [状态机 T-O2] | [API §4.1] |

**来源区分**：
- WP 自动接入：来源列显示"WP 自动"，站点列显示来源站点，自动进入 pending_review
- WhatsApp：来源列显示"WhatsApp"，站点列为空，先进入 draft，需手动提交 `[v2]`
- 手动录入：来源列显示"手动录入"，站点列为空，先进入 draft，需手动提交

**权限差异**：
- 老板：全部操作 [权限 R1-P6]
- 运营：可查看、可手动录入、可领取审核 [权限 R2-P6]
- 分销商：仅看到自己站点的订单 [权限 R3-P6]

**关键交互**：

| 操作 | 触发条件 | 结果 | 引用 |
|------|---------|------|------|
| 领取审核 | 状态 = pending_review | 状态 → reviewing，锁定给当前操作人 [状态机 T-O4] | [API §4.3] |
| 审核通过 | 状态 = reviewing | 状态 → pending_handover [状态机 T-O5] | [API §4.3] |
| 驳回 | 状态 = reviewing | 状态 → returned，需填写驳回原因 [状态机 T-O6] | [API §4.3] |
| 拆单 | 状态 = reviewing + 多供应商 | 展开拆单预览 → 确认后生成子订单 | [API §4.4] |
| 确认交接 | 状态 = pending_handover | 状态 → handed_over [状态机 T-O8] | [API §4.3] |
| 标记签收 | 状态 = handed_over | 状态 → delivered [状态机 T-O9] | [API §4.3] |
| 标记拒收 | 状态 = handed_over | 状态 → rejected → 自动创建拒收任务 [状态机 T-O10] | [API §4.3] |
| (系统) 进入待结算 | 状态 = delivered | 状态 → pending_settlement [状态机 T-O11，系统自动] | [API §4.3] |
| (系统) 进入待结算 | 状态 = rejected，拒收任务已关闭 | 状态 → pending_settlement [状态机 T-O12，系统自动] | [API §4.3] |
| 结算确认 | 状态 = pending_settlement | 状态 → closed [状态机 T-O13] | [API §4.3] |
| 标记异常 | 任意非终态 | 状态 → exception [状态机 T-O14] | [API §4.3] |
| 解除异常 | 状态 = exception | 恢复到异常前状态 [状态机 T-O15] | [API §4.3] |

**拆单逻辑**：
- 系统自动检测商品明细中的供应商数量，若 > 1 则显示拆单提示
- 拆单按供应商分组，每个子单独立选择履约路由
- 拆单后父订单状态锁定，子订单各自独立走交接→结果流程
- 父订单列表中显示"已拆单（父）"标记，点击可展开子单列表
