## 三、AI 的行为准则（对 AI 的约束，你可以监督）

### 不清楚时，问再动

AI 遇到以下情况，**必须先问，不得猜测**：

- 任务描述有歧义（"改一下那个按钮"→ 哪个按钮？）
- 涉及不可逆操作（删除数据、修改 migration）
- 两种实现方案各有取舍

问题格式必须是**封闭式**，不增加你的认知负担：

```
✅ 正确：这次修的是列表页筛选还是详情页筛选？[列表页/详情页]
✅ 正确：保存后是跳转详情页还是留在当前页？[跳转/留在当前页]
❌ 错误：你希望怎么处理这个功能？（开放式，让你思考）
❌ 错误：直接猜测并实现（不经确认）
```

### 完成任务后，自动做这三件事

每次完成一个功能或修复，AI **不需要你提醒**，自动：

1. 更新 `docs/CURRENT_STATUS.md`（任务状态 + 如有技术债则登记）
2. 如果发现了新的代码规律，更新对应的 `.cursor/rules/` 文件
3. 用一句话汇报："✅ 已完成 Bug-07，CURRENT_STATUS.md 已更新，可以测试：[具体操作步骤]"

### 对话变长时，主动提醒

当对话轮数超过 15 轮或内容变复杂，AI 主动说：
> "建议开新对话以保持准确性，我来生成交接指令，你直接粘贴给新对话即可。"

### 功能测试场景速查

说 **`测试`** 后，AI 会根据当前修复进度给出精确步骤。
以下是标准场景，供参考：

#### 场景 A：商品列表基础功能
```
1. 打开 http://localhost:5173/products
2. 验证：表格正常显示，有"统计条"（上方数字）
3. 点击列头"更新时间"：验证排序有效（顺序变化）
4. 在搜索框输入商品名：验证搜索过滤有效
```

#### 场景 B：受众标签筛选（Bug-07）
```
1. 商品列表页 → 点击"她用"chip
2. 验证：列表只显示 for_her 商品
3. 点击"他用"：只显示 for_him 商品
4. 再次点击取消：恢复显示全部
```

#### 场景 C：统计条快速过滤（Bug-08）
```
1. 统计条点击"未分类"数字
2. 验证：列表只显示无品类商品
3. 点击"缺图片"：只显示 image_urls 为空的商品
4. 点击"缺阿语"：只显示 title_ar 为空的商品
```

#### 场景 D：AI 辅助上线（Bug-02/03/04）
```
1. 点击"AI辅助上线"
2. Step1：上传1张图片 → 点击"AI分析"
3. Step2：验证品类下拉是从数据库加载的（非硬编码英文）
4. Step3：点击"保存为草稿"
5. 跳转详情页后验证：
   ✓ 图片可见（非空）
   ✓ 状态为"下架/草稿"（非"上架"）
   ✓ 品类显示正确（非"未分类"）
```

#### 场景 E：手动新增（Bug-01）
```
1. 商品列表页 → 点击"手动新增"
2. 验证：页面标题是"手动新增商品"（不是"AI辅助上线"）
3. 填写必填字段 → 保存
4. 跳转详情页，字段展示正确
```

#### 场景 F：供应商绑定（Bug-05）
```
1. 进入任意商品详情页 → 点击"供应商"Tab
2. 验证：有"添加供应商SKU"按钮
3. 点击 → 填写弹窗 → 保存
4. 验证：列表刷新，新增记录可见
5. 商品的 availability 变为"有货"
```

## 五、飞轮图（简化版）

```
你说口令
   ↓
AI 读上下文（自动）
   ↓
AI 不确定 → 封闭式问题 → 你回答 是/否
   ↓
AI 执行
   ↓
你在浏览器测试（AI 给步骤）
   ↓
发现问题 → "这个不对：XX"
没问题   → "收尾"
   ↓
AI 自动更新文档
   ↓
飞轮转一圈，规则更完善，下次更快
```
