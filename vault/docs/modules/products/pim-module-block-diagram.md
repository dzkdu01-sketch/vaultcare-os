## 2) 核心数据对象（对应三层模型）

```text
MasterSKU（全局真相）
  - master_code (不可变)
  - title / category / tags / images / descriptions ...

SiteListing（站点视图：7站点）
  - master_code + site_id
  - 覆盖价/可见性/库存等

WPSyncMapping（同步映射）
  - master_code + site_id
  - remote_product_id / sync_status / retry_count / last_error ...
```

---

## 3) 审核员操作闭环（最小动作集）

```text
选队列条目 → 处理红黄项（低置信/素材问题） → 保存草稿 → 提交审核 → 审核通过 → 批量推送

快捷键：
  A 通过下一条
  R 驳回（必填原因）
  S 保存
  N/P 切换
```

---

## 4) 关键约束（必须在实现中体现）

- `master_code`：唯一跨系统业务主键，**不可变**
- AI：只输出建议 + 置信度，**不自动发布**
- 7 站点：都支持覆盖价字段
- 推送失败：重试上限 `3`，超限进入老板异常队列
- 素材质检：`pass / auto-fixed / manual-required` 分层，`manual-required` 阻断提交
