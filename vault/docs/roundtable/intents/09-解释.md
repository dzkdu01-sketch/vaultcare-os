# 意图：/解释

**别名：** `/解读` `/说明` `/explain`

**触发时机：** 理解代码/逻辑/概念

---

## 作用

用易懂的方式解释复杂代码、业务逻辑或技术概念。

---

## 执行流程

```
1. 询问解释对象（文件/函数/概念）
2. 选择解释深度（概览/详细/逐行）
3. 输出解释 + 示例
4. 询问是否需要进一步解释
```

---

## 解释深度选项

| 深度 | 适用场景 | 输出长度 |
|------|----------|----------|
| 概览 | 快速了解 | 5-10 行 |
| 详细 | 深入理解 | 20-30 行 |
| 逐行 | 完全掌握 | 50+ 行 |

---

## 示例

**用户：** `/解释 @ProductList.tsx`

**AI：**
```
---
## ProductList.tsx 功能解释

**核心职责：** 商品列表页面（表格 + 卡片双视图）

**主要流程：**
1. 加载商品数据（useProductList hook）
2. 根据屏幕宽度切换视图（<768px 卡片视图）
3. 支持筛选/排序/批量操作

**关键状态：**
- viewMode: 'table' | 'card'
- filters: 筛选条件
- selectedIds: 批量操作选中的 ID

**用户交互：**
- 双击价格 → 行内编辑
- 点击统计条 → 快速筛选
- 勾选 + 批量按钮 → 批量操作

---
需要我详细解释某个部分吗？
```
