# 意图：/检查

**别名：** `/健康检查` `/check`

**触发时机：** 检查项目整体状态

---

## 作用

全面扫描项目健康度，发现潜在问题，给出改进建议。

---

## 执行流程

```
1. 自动扫描：
   - 待修复 Bug
   - 代码规范问题
   - 文档更新状态
   - 测试覆盖率
2. 输出检查报告
3. 建议优先级
```

---

## 检查清单

### Bug 状态
- [ ] P0 Bug 数量 = 0
- [ ] P1 Bug 数量 < 3

### 代码质量
- [ ] 无高风险审查问题
- [ ] 代码规范符合

### 文档状态
- [ ] current-status.md 已更新
- [ ] 模块 spec 同步

### 测试覆盖
- [ ] 核心功能有测试
- [ ] 测试通过率 100%

---

## 示例

**用户：** `/检查`

**AI：**
```
---
## 项目健康检查

### ✅ 正常项
- P0 Bug 数量 = 0
- 代码规范符合
- 测试通过率 100%

### ⚠️ 警告项
- P1 Bug 数量 = 3（建议<3）
- 文档更新延迟（>24 小时）

### ❌ 异常项
- 供应商模块无测试覆盖

---
**建议优先级：**
1. 补充供应商模块测试
2. 修复 P1 Bug
3. 更新文档
```
