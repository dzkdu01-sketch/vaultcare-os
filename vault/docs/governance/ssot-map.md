## 2) 运行规则（强制）

1. 非 SSOT 文档禁止复制粘贴大段正文，只能写“1 句摘要 + 1 个权威链接”。
2. 每次开发闭环最多回填 3 处：`current-status`、`system-functional-matrix`、`consensus`。
3. 发现冲突时，以 SSOT 为准；24 小时内完成冲突文档修复或打弃用标记。
4. 所有治理范围文档需维护 `Owner` 与 `last_verified_at`；超过 14 天未更新即“待复核”。

## 3) 冲突处理流程

1. 识别冲突（例如 API 路径、状态机口径、验收结论）。
2. 定位冲突项所属信息类型。
3. 回到对应 SSOT 文件判定主口径。
4. 修复非 SSOT 文档：改为引用，或打弃用标记。
5. 在 `docs/roundtable/consensus.md` 记录一次“冲突收敛”决策。

## 4) 度量口径（面向当前目标）

- token 消耗：单次 `/开发` 启动时注入上下文字数、引用文档数、重复段落数。
- 交接时间：从接手到“可开始编码”的分钟数、首轮读取文档数量。

> 目标值（首周）：
> - token 消耗较基线下降 30% 以上
> - 交接时间较基线下降 35% 以上
