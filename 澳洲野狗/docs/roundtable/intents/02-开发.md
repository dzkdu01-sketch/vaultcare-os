# 意图：/开发

**别名：** `/功能` `/feat`

**触发时机：** 开发新功能

---

## 作用

结构化开发流程，确保需求清晰、约束明确、验收标准具体。

---

## 执行流程

```
1. 追问约束（如"不改后端"）
2. 追问验收标准（如"怎么算完成"）
3. 输出流程设计/伪代码
4. 用户确认后生成代码
5. 输出 AI 自审查报告
6. 提示是否需要 /文档 更新
```

---

## 示例

**用户：** `/开发 供应商编辑功能`

**AI：**
```
收到。为了准确实现，我需要确认 3 个问题：

1. **约束**：是否可以修改后端 API？还是只能用现有接口？
2. **输入**：用户如何触发编辑？（按钮/双击/其他）
3. **验收**：编辑成功后需要什么反馈？

用户：1. 不改后端  2. 点击编辑按钮  3. 成功提示 + 列表刷新

AI：明白。先输出流程设计：

## 流程设计

1. 点击编辑按钮 → 弹出表单（预填当前数据）
2. 用户修改 → 点击保存
3. 调用 PUT /api/suppliers/{id}/
4. 成功：Toast 提示 + 刷新列表
5. 失败：显示错误信息

确认后可以开始写代码吗？
```
