# 意图：/审查

**别名：** `/review` `/代码审查`

**触发时机：** 审查代码质量

---

## 作用

自动扫描代码风险，输出审查报告，建议修复优先级。

---

## 执行流程

```
1. 询问审查范围（文件/模块/全量）
2. 自动扫描：
   - 接口规范符合性
   - 异常处理完整性
   - 安全隐患
   - 性能问题
3. 输出审查报告（按风险等级排序）
4. 建议修复优先级
```

---

## 审查清单

### 代码质量
- [ ] 接口定义符合现有规范
- [ ] 代码量 < 200 行（如超过已拆分）
- [ ] 无硬编码值（配置已提取）

### 异常处理
- [ ] 网络超时处理
- [ ] 数据校验
- [ ] 错误提示友好

### 安全
- [ ] 无 SQL 注入风险
- [ ] 无 XSS 风险
- [ ] 敏感数据不暴露

### 性能
- [ ] 无 N+1 查询
- [ ] 大数据量处理合理
- [ ] 缓存使用合理

---

## 示例

**用户：** `/审查 @ProductList.tsx`

**AI：**
```
---
## 代码审查报告

### 高风险（建议立即修复）
1. **N+1 查询**：循环内调用 API → 建议批量查询

### 中风险（本周内修复）
1. **错误处理缺失**：fetch 失败无提示
2. **硬编码值**：魔法数字 768（屏幕阈值）

### 低风险（择机修复）
1. **注释不足**：复杂逻辑无注释
2. **变量命名**：temp 等无意义命名

---
**修复优先级：** 高风险 → 中风险 → 低风险
```
