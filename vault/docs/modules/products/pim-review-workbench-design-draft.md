# 商品管理模块设计稿说明（审核员工作台）

> 版本：Draft v1  
> 场景：AI 辅助上新，审核员主操作，目标支持快速上线 100 个商品  
> 关联：`docs/modules/products/pim-v1-api-contract.md`

---

## 1. 设计目标

- 让审核员在低认知负担下完成高吞吐上新任务
- 保持数据规范：`master_code` 不可变、字段按三层模型落地
- AI 只做建议，不自动发布；人工确认后生效
- 与现有审核机制一致：拒绝可改后重提，异常由老板兜底

---

## 2. 页面信息架构

页面采用 **三栏布局 + 顶部批量操作 + 底部快捷键提示**：

1. **左栏：待审核队列**
   - 显示批次、SKU、风险标签、处理状态
   - 支持按风险筛选（低置信、疑似水印、疑似中文、缺图）
2. **中栏：商品编辑区**
   - 核心字段：标题、分类、标签、价格、短描述、图片列表
   - 字段旁 AI 按钮：优化标题、卖点提炼、竞品链接填充、一键补全
3. **右栏：素材质检区**
   - 主图/详情图预览
   - 质量分与问题清单
   - 状态分层：`pass` / `auto-fixed` / `manual-required`
4. **顶部：批量操作条**
   - 批量应用分类、标签、价格策略、批量提交审核
5. **底部：快捷键提示**
   - `A` 通过下一条、`R` 驳回、`S` 保存、`N/P` 切换

---

## 3. 功能模块说明

### 3.1 审核队列模块

- 以批次组织任务（支持 100 条分批推进）
- 卡片显示最小必要信息：`batch_id`、`master_code`、风险标签、处理状态
- 默认按“高风险优先”排序，减少返工

### 3.2 商品编辑模块

- 仅展示审核员高频核心字段，减少长表单疲劳
- 所有 AI 建议均先进入草稿态，不直接覆盖已确认字段
- 保存策略：
  - `保存`：仅保存草稿
  - `提交审核`：触发规则校验，校验通过才可提交

### 3.3 AI 辅助模块（就地按钮）

- **优化标题**：输出规范化标题模板（类目+核心卖点+规格）
- **卖点提炼**：生成 3-5 条标准卖点
- **竞品链接填充**：解析 URL，提取可用字段写入建议区
- **一键补全**：合并 AI 结果补齐缺失字段

约束：AI 结果必须携带 `confidence_by_field`，低置信字段高亮人工确认。

### 3.4 素材质检模块

- 上传后自动预检：
  - 分辨率/比例
  - 格式统一性
  - 疑似水印
  - 疑似中文残留
- 自动标准化（可配置）：
  - 主图：`1:1`
  - 详情图：`4:5`
- 输出：
  - 质量分（0-10）
  - 问题清单
  - 可执行建议（重传/自动修复/人工确认）

### 3.5 批量操作模块

- 支持对已勾选条目批量应用：
  - 分类
  - 标签
  - 价格策略
- 批量提交前展示摘要确认（数量、覆盖字段、风险项）

---

## 4. 关键交互流程

### 流程 A：单条快速审核

1. 队列选中条目  
2. 中栏自动加载字段与 AI 建议  
3. 审核员点“优化标题/卖点提炼”微调  
4. 右栏确认素材质量  
5. `A` 通过并切换下一条

### 流程 B：批量提效

1. 队列筛选同类商品  
2. 批量应用分类/标签/价格策略  
3. 抽样复核  
4. 批量提交审核  

### 流程 C：异常处理

1. 条目标记 `manual-required` 或推送失败超限  
2. 进入异常队列  
3. 老板处理后回写状态  

---

## 5. 状态与反馈设计

- 审核状态：`draft` / `pending_review` / `approved` / `rejected`
- 质检状态：`pass` / `auto-fixed` / `manual-required`
- 同步状态：`pending` / `syncing` / `success` / `failed` / `retrying`

反馈规则：

- 状态显示使用“颜色 + 文本”双通道（不只靠颜色）
- 失败文案必须业务化，例如：
  - “主图分辨率不足，建议重传或启用自动修复”
  - “竞品链接解析失败，请检查链接可访问性”

---

## 6. 可用性与无障碍要求

- 可点击区域最小 `44x44`
- 支持全键盘操作（含焦点可视）
- 关键按钮统一位置，避免跳动
- 列表与表单切换不丢失未保存内容

---

## 7. 与现有架构关系（影响评估）

低影响复用：

- `MasterSKU` / `SiteListing` / `WPSyncMapping`
- 审核与推送基础状态机

新增能力：

- 审核员工作台页面
- AI 就地辅助接口
- 素材质检与标准化流水线

总体评估：**中等改动、收益高**，适合 MVP 阶段。

---

## 8. MVP 验收标准（设计视角）

- 审核员可在一个页面完成“看队列-改字段-看素材-提交”
- 批量操作可覆盖分类/标签/价格策略
- AI 按钮可在单条录入页直接使用
- 素材问题可自动检测并输出可执行建议
- 100 条任务可按批次连续处理，不依赖多页面跳转

## 9. 字段级控件清单（研发对齐版）

### 9.1 左栏：待审核队列筛选与列表

| 区域 | 控件名 | 类型 | 必填 | 规则/说明 |
|---|---|---|---|---|
| 筛选区 | `batch_id` | 下拉 | 否 | 默认“全部批次” |
| 筛选区 | `risk_tags[]` | 多选 | 否 | `low_confidence` / `watermark` / `cn_text` / `missing_image` |
| 筛选区 | `review_status[]` | 多选 | 否 | `draft` / `pending_review` / `rejected` |
| 列表项 | `master_code` | 文本 | 是 | 只读，不可编辑 |
| 列表项 | `title_snapshot` | 文本 | 是 | 过长截断，hover 显示全量 |
| 列表项 | `risk_badges[]` | 标签 | 否 | 颜色+文字双显示 |
| 列表项 | `asset_status` | 标签 | 是 | `pass` / `auto-fixed` / `manual-required` |

### 9.2 中栏：商品编辑区字段

| 字段 | 控件类型 | 必填 | 校验规则 | AI能力 |
|---|---|---|---|---|
| `master_code` | 只读文本 | 是 | 不可变，不允许编辑 | 无 |
| `title` | 单行输入 | 是 | 10-120 字符 | `优化标题` |
| `primary_category` | 单选下拉 | 是 | 必须来自品类字典 | `竞品链接填充`可建议 |
| `tags[]` | 多选标签 | 是 | 至少 1 个，最多 8 个 | `卖点提炼`可建议 |
| `site_regular_price` | 数字输入 | 是 | >0，最多两位小数 | `竞品链接填充`可建议 |
| `site_sale_price` | 数字输入 | 是 | >0 且 <= regular_price | `竞品链接填充`可建议 |
| `short_description` | 多行输入 | 是 | 20-240 字符 | `卖点提炼`、`一键补全` |
| `description` | 富文本/多行 | 否 | 最多 2000 字符 | `一键补全` |
| `image_urls[]` | 图片列表 | 是 | 至少 1 张主图 | `一键补全`可补链接 |
| `video_urls[]` | 链接列表 | 否 | URL 格式校验 | `竞品链接填充`可建议 |

### 9.3 单条页就地 AI 按钮行为

| 按钮 | 输入 | 输出 | 回填策略 | 失败提示 |
|---|---|---|---|---|
| 优化标题 | 当前标题+分类+标签 | `title`, `confidence` | 先进入建议态，人工确认后替换 | “标题优化失败，请重试或手动编辑” |
| 卖点提炼 | 标题+短描+分类 | `selling_points[]` | 写入建议区，可一键合并到短描 | “卖点提炼失败，建议手动补充” |
| 竞品链接填充 | `competitor_url` | `suggested_fields`, `warnings[]` | 按字段勾选回填 | “链接不可访问或解析失败” |
| 一键补全 | 当前缺失字段 | `suggested_fields`, `confidence_by_field` | 仅填空，不覆盖已确认字段 | “补全失败，请检查网络后重试” |

### 9.4 右栏：素材质检区字段

| 字段 | 类型 | 来源 | 规则 |
|---|---|---|---|
| `quality_score` | 数值(0-10) | 质检服务 | `<7` 标红，`7-8.9` 标黄，`>=9` 标绿 |
| `issues[]` | 列表 | 质检服务 | 典型值：`size_inconsistent` / `format_mixed` / `suspected_watermark` / `suspected_cn_text` |
| `asset_status` | 枚举 | 质检服务 | `pass` / `auto-fixed` / `manual-required` |
| `normalized_main_image` | URL | 标准化服务 | 主图统一比例 `1:1` |
| `normalized_detail_images[]` | URL数组 | 标准化服务 | 详情图统一比例 `4:5` |

### 9.5 顶部批量操作字段

| 动作 | 入参字段 | 前置条件 | 结果 |
|---|---|---|---|
| 批量应用分类 | `item_ids[]`, `primary_category` | 至少选择 2 条 | 更新草稿并记录审计 |
| 批量应用标签 | `item_ids[]`, `tags[]` | 至少选择 2 条 | 覆盖标签或合并标签（可配置） |
| 批量价格策略 | `item_ids[]`, `pricing_rule` | 站点覆盖价已启用 | 生成各站点价格草稿 |
| 批量提交审核 | `item_ids[]` | 全部通过字段校验 | 状态改为 `pending_review` |

### 9.6 底部快捷键映射

| 快捷键 | 动作 | 安全规则 |
|---|---|---|
| `A` | 通过并下一条 | 若有未保存改动先弹确认 |
| `R` | 驳回 | 必填驳回原因 |
| `S` | 保存草稿 | 保存成功后显示 toast |
| `N` / `P` | 下一条 / 上一条 | 不丢失本条未提交草稿 |

### 9.7 提交前阻断校验（必须全部通过）

1. `master_code` 存在且未被修改  
2. `title`、`primary_category`、`tags`、`site_regular_price`、`site_sale_price`、`short_description` 非空  
3. `site_sale_price <= site_regular_price`  
4. 至少存在 1 张可用主图  
5. `asset_status != manual-required`（若为该状态，必须先处理或人工确认）

## 10. 竞品链接填充（细化规范）

### 10.1 可抓字段白名单

- `title`
- `short_description`
- `description`
- `primary_category_suggestion`
- `tags_suggestion[]`
- `regular_price_suggestion`
- `sale_price_suggestion`
- `image_urls[]`
- `video_urls[]`
- `specs_raw`

禁止写入字段：

- `master_code`
- 审核/同步状态字段
- 映射 ID 与系统审计字段

### 10.2 页面交互规则

1. 输入 `competitor_url` 后触发“竞品链接填充”。  
2. 结果进入“建议区”，不直接覆盖当前已确认值。  
3. 每个字段支持“采纳/忽略”。  
4. 低置信字段（<0.70）默认不勾选并高亮提示。  
5. 点击“应用建议”前展示覆盖摘要（将变更的字段列表）。  

### 10.3 置信度可视规则

- `>=0.85`：绿色，可一键采纳
- `0.70-0.84`：黄色，建议人工复核
- `<0.70`：红色，仅参考

### 10.4 失败与降级体验

- `FETCH_TIMEOUT`：提示“链接读取超时，请稍后重试”
- `PARSE_FAILED`：提示“页面结构不支持自动提取，请手动录入”
- `URL_BLOCKED`：提示“该站点不在可抓取范围”
- 若仅提取到部分字段，仍返回可用字段并标注“部分成功”
