> 覆盖 Gate 4 验收标准 AC-4：用户能看到"为什么被卡住"。

## 1. 设计原则

1. **即时可见** — 用户不需要猜测下一步该做什么，系统主动告知阻塞原因
2. **分层展示** — 列表页给摘要，详情页给完整信息，避免信息过载
3. **可操作** — 每个阻塞提示直接指向需要补充的字段或操作，而非泛泛提示

## 2. 三层门禁可见性架构

```
┌─ 第一层：列表页摘要 ──────────────────────────────────────┐
│                                                           │
│  场景：页面 1 商品工作台首页                                │
│                                                           │

│  鼠标悬停 → 浮层显示一句话原因：                            │
```

## 3. 各页面门禁可见性清单

| 页面 | 可见性组件 | 门禁类型 | 展示方式 |
|------|-----------|---------|---------|
| 页面 1 | 门禁卡点列 | 商品门禁 G-P1~G-P5 | 列摘要 + 悬停浮层 |
| 页面 2 | 状态进度条 + 按钮状态 | 商品门禁 G-P1~G-P5 | 进度条 + 阻塞卡片 + 按钮灰置 |
| 页面 4 | 前置依赖提示 | 页面依赖（页面 3 数据） | 引导提示条 |
| 页面 4 | 发布按钮状态 | 站点 webhook 配置状态 | 按钮灰置 + 悬停提示 |
| 页面 7 | 状态进度条 | 订单状态流转 | 进度条（无门禁阻塞，但显示当前位置） |
| 页面 7 | 拆单提示 | 多供应商检测 | ⚠ 提示条 |
| 页面 8 | 关闭条件提示 | 结算状态检查 | 按钮灰置 + 悬停提示 |

## 4. 门禁提示文案规范

所有门禁提示遵循统一格式：

```
[门禁编号] [门禁名称]
→ [具体缺失项1]
→ [具体缺失项2]
```

示例：
- `G-P2 身份门禁 → 供应商未关联`
- `G-P3 内容门禁 → 划线价或实际售价未填写`
- `G-P4 素材门禁 → 主图未确认`
- `页面依赖 → 请先在「分销商与站点管理」中配置至少一个启用的分销商和站点`
- `关闭条件 → 结算状态仍为"未结算"，需先确认结算方式`
