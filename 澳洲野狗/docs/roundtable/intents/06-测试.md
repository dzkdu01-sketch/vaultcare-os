# 意图：/测试

**别名：** `/用例` `/test`

**触发时机：** 生成测试用例

---

## 作用

生成覆盖正常/异常/边界的测试用例，确保功能质量。

---

## 执行流程

```
1. 询问测试对象（功能/模块）
2. 追问测试场景（正常/异常/边界）
3. 生成测试用例列表
4. 询问是否需要自动化测试脚本
```

---

## 测试用例模板

```markdown
### 正常路径
- [ ] 用例 1：标准输入，期望输出
- [ ] 用例 2：...

### 异常路径
- [ ] 用例 1：网络超时
- [ ] 用例 2：数据校验失败
- [ ] 用例 3：权限不足

### 边界条件
- [ ] 用例 1：空输入
- [ ] 用例 2：最大值
- [ ] 用例 3：最小值
```

---

## 示例

**用户：** `/测试 供应商编辑功能`

**AI：**
```
---
## 测试用例：供应商编辑功能

### 正常路径
- [ ] 编辑 cost_price，保存成功，列表显示新价格
- [ ] 编辑 stock_status，保存成功，状态更新
- [ ] 编辑后取消，数据不保存

### 异常路径
- [ ] 网络超时，显示友好错误提示
- [ ] 价格为负数，校验失败
- [ ] 并发编辑，后保存者提示冲突

### 边界条件
- [ ] 价格 = 0，允许
- [ ] 价格 = 999999999，允许
- [ ] 特殊字符输入，正确转义

---
需要我生成自动化测试脚本吗？（Jest/Pytest）
```
